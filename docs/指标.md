#1 全部指标和计算公式

> **统一记号（适用于涉及 spot/futures 的指标）**
> - 主市场：`F`（Futures，币安 ETH 永续/合约）
> - 归因/确认市场：`S`（Spot，币安 ETHUSDT 现货）
> - 统一数量口径：`qty_eth`（ETH base qty）或 `notional_usdt`
> - 主动方向：`a_i ∈ {+1,-1}`（主动买=+1，主动卖=-1）
> - Binance `m` 字段映射：`m=true → a_i=-1`，`m=false → a_i=+1`
>
> **文档目标**
> - 列出全部指标与计算公式
> - 在每个指标处明确所需市场数据（合约/现货）
> - 双市场场景下保持“合约主判定 + 现货确认/归因”的精度设计

## 1) price_volume_structure

### 数据源要求（市场）
- **主计算市场**：合约（F）
- **现货数据（S）**：不需要（本指标按合约成交结构计算）
- **必需数据类型**：合约逐笔成交（trades，含主动方向）

> 本容器统一输出：
> - Value Area（`VAL / VAH / ValueAreaLevels`）
> - Volume Profile（分价位 `buy/sell/total/delta`）
> - POC（`poc_price / poc_volume`）
> - HVN / LVN
> - Volume Dry-up（窗口级）

### 统计窗口
- 当前 1m bar 快照（顶层字段，保持原有结构）
- **多窗口**：`w ∈ {15m, 4h, 1d, 3d}`
  - 每个窗口将最近 `w` 分钟内的所有 1m bar 的分价位成交量聚合后计算完整 PVS
  - 输出格式：顶层新增 `by_window` 字典，键为 `"15m"` / `"4h"` / `"1d"` / `"3d"`
  - **严格约束**：`price_volume_structure.payload.by_window` 仅允许以上 4 个窗口，不输出 `1h` 等其他窗口

### 输入
- `trades`
  - `timestamp`
  - `price`
  - `qty`
  - `aggressor_side`（若交易所直接提供）
  - 或 Binance `m` 字段（映射为主动方向）
- `tick_size` / `price_bin_size = Δp`
- `value_area_ratio`（固定默认 `0.70`）
- `smooth_kernel = K`（HVN/LVN 平滑核，固定 `[1,2,1]/4`）
- `hvn_prominence_threshold = \gamma_{hvn}`（固定 `3%` of window total volume）
- `lvn_prominence_threshold = \gamma_{lvn}`（固定 `1%` of window total volume）
- `dryup_lookback = N`（固定 `30` 个同尺寸窗口周期，不足时取实际可用数量，最少需 5 个）
- `dryup_z_threshold = k_v`（固定 `1.0`）
- `key_level_test_flag`（若未提供，固定实现 `KeyTest_w[t]\equiv 1`）
- `eps = 1e-12`

> **主动方向映射（Binance）**
> - `m = true` ⇒ 主动卖（`a_i=-1`）
> - `m = false` ⇒ 主动买（`a_i=+1`）

### 输出字段
- 顶层（1m 当前 bar 快照）：`bar_volume, poc_price, poc_volume, val, vah, value_area_levels, hvn_levels, lvn_levels, volume_zscore, volume_dryup, levels[]`
- `by_window["15m" | "4h" | "1d" | "3d"]`：每个窗口包含相同字段集合，另加 `window_bars_used`（实际参与聚合的 1m bar 数量）
- 语义约束：
  - 顶层 `1m` 快照仅表示 current bar sidecar
  - 持仓后判断优先读取 `by_window`

### 计算公式

#### 1.1 分价位买方 / 卖方成交量（主动方向拆分）
设窗口 \(w\) 在时刻 \(t\) 的成交集合为 \(I_w(t)\)，价格离散到价位 \(x\)。

\[
B_w[t,x]=\sum_{i\in I_w(t),\,p_i=x,\,a_i=+1} q_i
\]
\[
S_w[t,x]=\sum_{i\in I_w(t),\,p_i=x,\,a_i=-1} q_i
\]

#### 1.2 Volume Profile（分价位总成交量）
\[
V_w[t,x]=B_w[t,x]+S_w[t,x]
\]

对应 `levels[]` 字段：
- `buy_volume = B_w[t,x]`
- `sell_volume = S_w[t,x]`
- `volume = V_w[t,x]`

#### 1.3 分价位 Delta
\[
\Delta^{level}_w[t,x]=B_w[t,x]-S_w[t,x]
\]

对应 `levels[]` 字段：
- `delta = \Delta^{level}_w[t,x]`

#### 1.4 窗口总成交量（bar volume）
\[
Vol_w[t]=\sum_x V_w[t,x]
\]

对应顶层字段：
- `bar_volume = Vol_w[t]`

#### 1.5 POC（并入顶层）
\[
POC_w[t]=\arg\max_x V_w[t,x]
\]
\[
POCVol_w[t]=V_w[t,POC_w[t]]
\]

对应：
- `poc_price = POC_w[t]`
- `poc_volume = POCVol_w[t]`

#### 1.6 Value Area（并入顶层）
总成交量：
\[
Vol_w[t]=\sum_x V_w[t,x]
\]

从 POC 向两侧扩展构造价值区价位集合 \(\mathcal{A}_w[t]\)，直到累计成交量覆盖 `value_area_ratio`：
\[
\sum_{x\in\mathcal{A}_w[t]}V_w[t,x]\ge value\_area\_ratio\cdot Vol_w[t]
\]

输出区间：
\[
VAL_w[t]=\min(\mathcal{A}_w[t]),\qquad VAH_w[t]=\max(\mathcal{A}_w[t])
\]

价值区价位数组：
\[
ValueAreaLevels_w[t]=\{(x,V_w[t,x])\mid x\in\mathcal{A}_w[t]\}
\]

逐价位是否处于价值区：
\[
isInVA_w[t,x]=\mathbf{1}\{x\in\mathcal{A}_w[t]\}
\]

对应：
- `val = VAL_w[t]`
- `vah = VAH_w[t]`
- `value_area_levels = ValueAreaLevels_w[t]`
- `levels[].is_in_value_area = isInVA_w[t,x]`

#### 1.7 HVN / LVN（分价位节点）
先平滑：
\[
\widetilde V_w[t,x]=(V_w[t,\cdot]*K)(x)
\]

HVN 条件：
\[
x\in HVN_w[t]\iff \widetilde V_w[t,x]>\widetilde V_w[t,x-\Delta p]\land \widetilde V_w[t,x]>\widetilde V_w[t,x+\Delta p]\land Prominence(x)\ge \gamma_{hvn}
\]

LVN 条件：
\[
x\in LVN_w[t]\iff \widetilde V_w[t,x]<\widetilde V_w[t,x-\Delta p]\land \widetilde V_w[t,x]<\widetilde V_w[t,x+\Delta p]\land Prominence(x)\ge \gamma_{lvn}
\]

顶层数组：
\[
\mathcal{H}_w[t]=\{x\mid x\in HVN_w[t]\},\qquad
\mathcal{L}_w[t]=\{x\mid x\in LVN_w[t]\}
\]

逐价位标记：
\[
isHVN_w[t,x]=\mathbf{1}\{x\in HVN_w[t]\},\qquad
isLVN_w[t,x]=\mathbf{1}\{x\in LVN_w[t]\}
\]

#### 1.8 Volume Dry-up（窗口级，固定并入顶层）
历史均值（不含当前）：
\[
\mu_w^{(N)}[t]=\frac{1}{N}\sum_{k=1}^{N}Vol_w[t-k]
\]

历史标准差（不含当前）：
\[
\sigma_w^{(N)}[t]=std(Vol_w[t-1],\ldots,Vol_w[t-N])
\]

成交量 z-score：
\[
zVol_w[t]=\frac{Vol_w[t]-\mu_w^{(N)}[t]}{\sigma_w^{(N)}[t]+\varepsilon}
\]

基础枯竭判定：
\[
DryUpBase_w[t]=\mathbf{1}\{zVol_w[t]\le -k_v\}
\]

关键位测试标记（若上层未提供则固定实现 \(KeyTest_w[t]\equiv 1\)）：
\[
VolumeDryUp_w[t]=DryUpBase_w[t]\cdot KeyTest_w[t]
\]

对应：
- `volume_zscore = zVol_w[t]`
- `volume_dryup = VolumeDryUp_w[t]`

---

### 多窗口聚合公式（1.9）

对窗口 \(w\in\{15m,4h,1d,3d\}\)，设窗口所含 1m bar 数量为 \(N_w\)（15/240/1440/4320），当前时刻 \(t\)。

窗口内聚合 profile（将各 1m bar 的分价位成交量相加）：
\[
B_w[t,x]=\sum_{k=0}^{N_w-1}B_{1m}[t-k,x], \quad
S_w[t,x]=\sum_{k=0}^{N_w-1}S_{1m}[t-k,x]
\]
其余字段（$V_w, \Delta_w, POC_w, VA_w, HVN_w, LVN_w$）按公式 1.2–1.7 对聚合 profile 计算。

Volume Dry-up 的历史样本为 **非重叠** 的前 N 个同尺寸窗口的 \(Vol_w\) 值：
\[
\mu_w^{(N)}[t] = \frac{1}{N}\sum_{i=1}^{N}Vol_w[t-i\cdot N_w],\quad
\sigma_w^{(N)}[t] = std(Vol_w[t-N_w],\ldots,Vol_w[t-N\cdot N_w])
\]
\[
zVol_w[t]=\frac{Vol_w[t]-\mu_w^{(N)}[t]}{\sigma_w^{(N)}[t]+\varepsilon}
\]

### 输出序列定义（窗口级）
对每个窗口周期 \(w\) 的每个 bar 结束时刻 \(t\) 输出：
\[
Record^{PVS}_w[t]=
\Big(
t,\ N_w^{used},\ Vol_w[t], zVol_w[t], VolumeDryUp_w[t], POC_w[t], POCVol_w[t], VAL_w[t], VAH_w[t], HVN_w[t], LVN_w[t],
\mathcal{A}_w[t],
\{(x,B_w,S_w,V_w,\Delta^{level}_w,isHVN,isLVN,isInVA)\}_x
\Big)
\]

## 2) footprint

### 数据源要求（市场）
- **主计算市场**：合约（F）
- **现货数据（S）**：不需要（本指标按合约逐笔足迹矩阵计算）
- **必需数据类型**：合约逐笔成交（trades，含主动方向）

> 本容器统一输出：
> - 足迹矩阵（每价位 buy/sell/delta/total）
> - 买卖失衡与失衡价格数组
> - Stacked Imbalances（连串失衡段与最大连串长度）
> - Unfinished Auction（顶/底部未完成拍卖）
> - **UA 在足迹价格区域显示**（`levels[]` 中给出 `ua_top_flag / ua_bottom_flag`）

### 输入
- trades（必须含主动方向线索）
  - `timestamp`
  - `price`
  - `qty`
  - `aggressor_side`
  - 或 Binance `m`（`buyer_is_maker`）
- `tick_size = Δp`
- `imbalance_ratio_threshold = \theta`（固定默认 `3.0`）
- `min_level_volume = v_min`（固定默认 `0` 或业务阈值）
- `min_consecutive_levels = n_stack`（固定默认 `3`）
- `unfinished_auction_min_qty = v^{ua}_{min}`（固定默认由业务给定）
- `lookback_length = 7d`
- `window ∈ {15m, 1h, 4h, 1d, 3d}`
- `eps = 1e-12`

> **Binance 主动方向映射**
> - `m = true` ⇒ 主动卖 ⇒ `a_i=-1`
> - `m = false` ⇒ 主动买 ⇒ `a_i=+1`

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）

### 计算公式

#### 2.1 主动方向（aggressor side）
\[
a_i=
\begin{cases}
-1,& m_i=\text{true}\\
+1,& m_i=\text{false}
\end{cases}
\]

#### 2.2 窗口与输出长度
统计窗口：
\[
w\in\{15m,1h,4h,1d,3d\}
\]
总输出长度固定：
\[
L=7d
\]
对每个 bar 结束时刻 \(t\)：
\[
I_w(t)=(t-w,\ t]
\]

#### 2.3 分价位足迹（核心矩阵）
\[
B_w[t,x]=\sum_{i\in I_w(t),\,p_i=x,\,a_i=+1} q_i
\]
\[
S_w[t,x]=\sum_{i\in I_w(t),\,p_i=x,\,a_i=-1} q_i
\]
\[
Footprint_w[t,x]=(S_w[t,x],\,B_w[t,x])
\]

#### 2.4 level 内字段
\[
\Delta^{level}_w[t,x]=B_w[t,x]-S_w[t,x]
\]
\[
Total^{level}_w[t,x]=B_w[t,x]+S_w[t,x]
\]

#### 2.5 当前窗口总 Delta / 总成交量
\[
\Delta_w[t]=\sum_x(B_w[t,x]-S_w[t,x])=\sum_{i\in I_w(t)}a_i q_i
\]
\[
Vol_w[t]=\sum_x(B_w[t,x]+S_w[t,x])
\]

对应：
- `window_delta = \Delta_w[t]`
- `window_total_qty = Vol_w[t]`

#### 2.6 开盘/收盘/最高/最低价位标记（显示在足迹价格区域）
若 bar 由 trades 聚合：
\[
O_w[t]=\text{窗口内第一笔成交价格},\quad C_w[t]=\text{窗口内最后一笔成交价格}
\]
\[
H_w[t]=\max_{i\in I_w(t)} p_i,\qquad L_w[t]=\min_{i\in I_w(t)} p_i
\]

逐价位标记：
\[
isOpen_w[t,x]=\mathbf{1}\{x=O_w[t]\},\quad
isClose_w[t,x]=\mathbf{1}\{x=C_w[t]\}
\]
\[
isHigh_w[t,x]=\mathbf{1}\{x=H_w[t]\},\quad
isLow_w[t,x]=\mathbf{1}\{x=L_w[t]\}
\]

#### 2.7 失衡标记（对角线错一格匹配）
买方失衡：
\[
BuyImb_w[t,x]=
\mathbf{1}\left(
\frac{B_w[t,x]}{S_w[t,x-\Delta p]+\varepsilon}\ge \theta
\land B_w[t,x]\ge v_{\min}
\right)
\]

卖方失衡：
\[
SellImb_w[t,x]=
\mathbf{1}\left(
\frac{S_w[t,x]}{B_w[t,x+\Delta p]+\varepsilon}\ge \theta
\land S_w[t,x]\ge v_{\min}
\right)
\]

失衡价格数组：
\[
\mathcal{I}^{buy}_w[t]=\{x\mid BuyImb_w[t,x]=1\}
\]
\[
\mathcal{I}^{sell}_w[t]=\{x\mid SellImb_w[t,x]=1\}
\]

#### 2.8 Stacked Imbalances（连串失衡）
买方连串长度最大值：
\[
StackRun^{buy}_w[t]=\max\left\{n:\exists x,\ \prod_{k=0}^{n-1} BuyImb_w[t,x+k\Delta p]=1\right\}
\]

卖方连串长度最大值：
\[
StackRun^{sell}_w[t]=\max\left\{n:\exists x,\ \prod_{k=0}^{n-1} SellImb_w[t,x-k\Delta p]=1\right\}
\]

窗口级布尔标记：
\[
StackedBuy_w[t]=\mathbf{1}\{StackRun^{buy}_w[t]\ge n_{stack}\},\quad
StackedSell_w[t]=\mathbf{1}\{StackRun^{sell}_w[t]\ge n_{stack}\}
\]

连串段输出（`buy_stacks[] / sell_stacks[]`）定义为满足长度 \(\ge n_{stack}\) 的所有极大连续区间。

#### 2.9 Unfinished Auction（未完成拍卖，且显示在 price levels）
顶端未完成拍卖：
\[
UA^{top}_w[t]=\mathbf{1}\{B_w[t,H_w[t]]>v^{ua}_{min}\land S_w[t,H_w[t]]>v^{ua}_{min}\}
\]

底端未完成拍卖：
\[
UA^{bot}_w[t]=\mathbf{1}\{B_w[t,L_w[t]]>v^{ua}_{min}\land S_w[t,L_w[t]]>v^{ua}_{min}\}
\]

窗口级汇总值：
\[
UA_w[t]=UA^{top}_w[t]-UA^{bot}_w[t]
\]

**显示在足迹价格区域（你要求）**：在 `levels[]` 中加入价位级标记
\[
uaTopFlag_w[t,x]=\mathbf{1}\{x=H_w[t]\}\cdot UA^{top}_w[t]
\]
\[
uaBottomFlag_w[t,x]=\mathbf{1}\{x=L_w[t]\}\cdot UA^{bot}_w[t]
\]

对应字段：
- 顶层：`ua_top`, `ua_bottom`, `unfinished_auction`
- `levels[]`：`ua_top_flag`, `ua_bottom_flag`

## 3) divergence（CVD 背离 / 顶背离 / 底背离，严谨版：去趋势 + 标准化 + 显著性）

### 数据源要求（市场）
- **主判定市场**：合约（F）
- **现货数据（S）**：**需要**（用于 CVD 背离归因与现货主导确认）
- **必需数据类型（F）**：合约 1m OHLCV + 合约逐笔成交（构造 1m Delta / 1m CVD）
- **必需数据类型（S）**：现货逐笔成交（构造 1m Delta / 1m CVD）

> **本版修订目标（相对旧版）**
> 1. **避免未来函数**：pivot 使用“对称定义 + 延迟确认”，事件仅在确认后输出  
> 2. **降低伪背离**：对价格/CVD 做 **去趋势 + 标准化**（CVD 非平稳问题）  
> 3. **提高可交易性**：加入 **显著性检验 / 最小效应量门槛**（ATR 与 robust-z）  
> 4. **保留双市场逻辑**：事件检测仍以合约为主，现货做归因与确认

> 统一收纳四类背离事件：
> - `bearish_divergence`（顶背离 / 常规看跌）
> - `hidden_bearish_divergence`（隐藏看跌）
> - `bullish_divergence`（底背离 / 常规看涨）
> - `hidden_bullish_divergence`（隐藏看涨）

### 计算公式（修订版）

#### 3.1 分钟级 Delta（1m）
对市场 \(x\in\{F,S\}\)（合约 / 现货）：
\[
\Delta^{x}_{1m}[m]=\sum_{i\in I^{x}_{1m}(m)} a_i q_i,\quad a_i\in\{+1,-1\}
\]

#### 3.2 分钟级 CVD（Rolling 7D，固定 1m 粒度）
\[
CVD^{7d,x}_{1m}[m]=\sum_{u\in(m-7d,\ m]} \Delta^{x}_{1m}[u]
\]
记：
\[
\widetilde{CVD}^{x}[m]\equiv CVD^{7d,x}_{1m}[m]
\]

> **说明（关键）**：直接使用 CVD 水平值做背离，容易受长期单边趋势影响产生伪背离；因此后续判定使用 **去趋势 + 标准化** 后的 CVD 特征。

#### 3.3 价格与 CVD 的去趋势（全因果实现）
定义价格工作变量（合约）：
\[
P[m]=\log(C[m])
\]

使用 **仅过去数据** 的局部趋势估计（可实现为滚动 OLS 或 EMA；以下以滚动 OLS 表示）：

价格趋势（窗口 \(n_p\)）：
\[
\hat P^{trend}[m]=\hat\beta^{P}_{0,m}+\hat\beta^{P}_{1,m}\cdot m
\quad\text{（在样本 } \{m-n_p+1,\ldots,m\}\text{ 上拟合）}
\]

CVD 趋势（对市场 \(x\) ，窗口 \(n_c\)）：
\[
\widehat{CVD}^{trend,x}[m]=\hat\beta^{C,x}_{0,m}+\hat\beta^{C,x}_{1,m}\cdot m
\quad\text{（在样本 } \{m-n_c+1,\ldots,m\}\text{ 上拟合）}
\]

去趋势残差：
\[
P^{dt}[m]=P[m]-\hat P^{trend}[m]
\]
\[
C^{dt,x}[m]=\widetilde{CVD}^{x}[m]-\widehat{CVD}^{trend,x}[m]
\]

#### 3.4 Robust 标准化（抗尖峰）
定义 rolling median 与 MAD（均只使用过去样本）：
\[
med_Z[m]=median(Z[m-n_z+1],\ldots,Z[m])
\]
\[
MAD_Z[m]=median\big(|Z[u]-med_Z[m]|:u\in[m-n_z+1,m]\big)
\]

robust z-score：
\[
rz(Z)[m]=\frac{Z[m]-med_Z[m]}{1.4826\cdot MAD_Z[m]+\varepsilon}
\]

应用到价格 / CVD（价格用合约，CVD 对 F/S 分别计算）：
\[
Z^{P}[m]=rz(P^{dt})[m]
\]
\[
Z^{C,F}[m]=rz(C^{dt,F})[m],\qquad Z^{C,S}[m]=rz(C^{dt,S})[m]
\]

> 若实现复杂度受限，可退化为 rolling mean/std z-score；但优先 robust z，以降低大单尖峰造成的误判。

#### 3.5 1m Swing High / Swing Low（**延迟确认版，无未来函数泄漏**）
仍使用对称 pivot 定义，但 **仅在右侧 \(k\) 根完成后确认**。  
对候选枢轴索引 \(m\)：

候选高点：
\[
CandSwingHigh[m]=\mathbf{1}\Big(H[m]>\max(H[m-k],\dots,H[m-1])\ \land\ H[m]\ge \max(H[m+1],\dots,H[m+k])\Big)
\]

候选低点：
\[
CandSwingLow[m]=\mathbf{1}\Big(L[m]<\min(L[m-k],\dots,L[m-1])\ \land\ L[m]\le \min(L[m+1],\dots,L[m+k])\Big)
\]

**确认时间**定义为：
\[
t^{conf}(m)=t(m+k)
\]

只有当当前时间 \(T\ge t^{conf}(m)\) 时，pivot 才可用于事件检测。  
因此策略可用 pivot 集合（截至时刻 \(T\)）：
\[
\mathcal{P}^{high}(T)=\{m\mid CandSwingHigh[m]=1,\ t^{conf}(m)\le T\}
\]
\[
\mathcal{P}^{low}(T)=\{m\mid CandSwingLow[m]=1,\ t^{conf}(m)\le T\}
\]

> **避免未来函数的关键**：可以记录 pivot 发生在 \(m\)，但事件的 `event_available_ts` 必须不早于相关 pivot 的确认时间（至少为 \(t(m+k)\)）。

#### 3.6 统计窗口覆盖的 1m 索引集合（事件发生时刻约束 + 可用时刻约束）
对统计窗口 \(w\) 在输出时刻 \(T\)：
\[
\mathcal{M}_w(T)=\{m\mid T-w < t(m)\le T\}
\]

窗口内已确认 pivot 集合：
\[
\mathcal{P}^{high}_w(T)=\mathcal{P}^{high}(T)\cap \mathcal{M}_w(T),\qquad
\mathcal{P}^{low}_w(T)=\mathcal{P}^{low}(T)\cap \mathcal{M}_w(T)
\]

#### 3.7 事件配对规则（新增最小间隔与质量约束）
对同侧 pivot 按时间排序，形成相邻候选对 \((t_1,t_2)\)，要求：
\[
G_{min}\le t_2-t_1\le G_{max}
\]

并建议加入最小价格腿长（避免微噪声）：
\[
\frac{|C[t_2]-C[t_1]|}{ATR^{(n_{atr})}_{1m}[t_2]+\varepsilon}\ge \eta_{leg}
\]
其中 \(\eta_{leg}\) 为最小效应量阈值（如 `0.5`）。

#### 3.8 顶背离 / 底背离的改进定义（使用去趋势 + 标准化后的 CVD）
为保证定义可交易且稳健，**价格比较仍使用原始枢轴价（H/L）**，但 CVD 比较使用 \(Z^{C,F}\)（合约 CVD 去趋势 robust-z）。

定义方向化差值（按 pivot 类型）：

高点类（top / bearish family）：
\[
\Delta P^{high}_e = H[t_2]-H[t_1]
\]
\[
\Delta C^{high,F}_e = Z^{C,F}[t_2]-Z^{C,F}[t_1]
\]

低点类（bottom / bullish family）：
\[
\Delta P^{low}_e = L[t_2]-L[t_1]
\]
\[
\Delta C^{low,F}_e = Z^{C,F}[t_2]-Z^{C,F}[t_1]
\]

同时定义 ATR 归一化价格变化（效应量）：
\[
zP^{high}_e=\frac{\Delta P^{high}_e}{ATR^{(n_{atr})}_{1m}[t_2]+\varepsilon}
\]
\[
zP^{low}_e=\frac{\Delta P^{low}_e}{ATR^{(n_{atr})}_{1m}[t_2]+\varepsilon}
\]

CVD 端使用已标准化序列差值作为效应量：
\[
zC^{high}_e=\Delta C^{high,F}_e,\qquad zC^{low}_e=\Delta C^{low,F}_e
\]

---

##### 3.8.1 顶背离（常规看跌，`bearish_divergence`）
\[
\Delta P^{high}_e \ge \epsilon_{price}
\quad\land\quad
\Delta C^{high,F}_e \le -\epsilon_{cvd,z}
\]
并通过显著性门槛（见 3.9）：
\[
SigPass_e=1
\]

解释：价格创更高高点，但合约 CVD（去趋势+标准化）未确认，形成更低高点。

##### 3.8.2 隐藏顶背离（隐藏看跌，`hidden_bearish_divergence`）
\[
\Delta P^{high}_e \le -\epsilon_{price}
\quad\land\quad
\Delta C^{high,F}_e \ge \epsilon_{cvd,z}
\]
且：
\[
SigPass_e=1
\]

##### 3.8.3 底背离（常规看涨，`bullish_divergence`）
\[
\Delta P^{low}_e \le -\epsilon_{price}
\quad\land\quad
\Delta C^{low,F}_e \ge \epsilon_{cvd,z}
\]
且：
\[
SigPass_e=1
\]

解释：价格创更低低点，但合约 CVD（去趋势+标准化）形成更高低点。

##### 3.8.4 隐藏底背离（隐藏看涨，`hidden_bullish_divergence`）
\[
\Delta P^{low}_e \ge \epsilon_{price}
\quad\land\quad
\Delta C^{low,F}_e \le -\epsilon_{cvd,z}
\]
且：
\[
SigPass_e=1
\]

> **阈值口径建议（重要）**
> - `\epsilon_price` 建议采用 **ATR 标准化门槛**，而非固定价格点数  
> - `\epsilon_{cvd,z}` 建议在 robust-z 序列上设置（如 `0.5 ~ 1.5`）

#### 3.9 显著性检验（新增，避免“看起来像背离”的噪声）
本版提供两种模式，默认 `threshold`，可选 `block_bootstrap`。

##### 3.9.1 Threshold 模式（默认，低延迟）
高点类事件：
\[
SigPass^{high}_e=
\mathbf{1}\{|zP^{high}_e|\ge z_p^{min}\ \land\ |zC^{high}_e|\ge z_c^{min}\}
\]

低点类事件：
\[
SigPass^{low}_e=
\mathbf{1}\{|zP^{low}_e|\ge z_p^{min}\ \land\ |zC^{low}_e|\ge z_c^{min}\}
\]

统一：
\[
SigPass_e=
\begin{cases}
SigPass^{high}_e,& \text{high pivots}\\
SigPass^{low}_e,& \text{low pivots}
\end{cases}
\]

##### 3.9.2 Block Bootstrap 模式（可选，统计更严谨）
对事件腿区间 \([t_1,t_2]\) 内的 1m 增量序列做块自举（保留自相关）：
- 价格增量：\(r_u=\log C[u]-\log C[u-1]\)
- CVD 增量：\(\Delta^{F}_{1m}[u]\)

构造块长度 \(b\) 的 bootstrap 样本（重复 \(B\) 次），计算模拟效应量分布，得到双侧 p 值：
\[
p^{price}_e = \Pr\big(|zP^{*}_e|\ge |zP_e|\big)
\]
\[
p^{cvd}_e = \Pr\big(|zC^{*}_e|\ge |zC_e|\big)
\]

显著性通过条件：
\[
SigPass_e=\mathbf{1}\{p^{price}_e\le \alpha \land p^{cvd}_e\le \alpha\}
\]

> 若追求实时性，推荐在线策略使用 `threshold`；离线回测/研究可启用 `block_bootstrap`。

#### 3.10 事件可用时刻（避免未来函数，必须落地）
对事件 \(e=(t_1,t_2)\)，设对应 pivot 索引为 \(m_1,m_2\)，则：
\[
event\_available\_ts(e)=\max\big(t^{conf}(m_1),\ t^{conf}(m_2)\big)
\]

若还需要等待额外确认 bar（如 `confirm_bars = c_f`）：
\[
event\_available\_ts(e)=\max\big(t^{conf}(m_1), t^{conf}(m_2)\big)+c_f\cdot 1m
\]

> 回测与实时交易都应使用 `event_available_ts` 作为事件生效时间，而不是 pivot 原始发生时间。

#### 3.11 强度分数（修订：加入显著性与归一化效应量）
定义价格与 CVD 效应量（按事件类型选择 high/low 对应值）：
\[
a_P(e)=
\begin{cases}
|zP^{high}_e|,& \text{high pivots}\\
|zP^{low}_e|,& \text{low pivots}
\end{cases}
\]
\[
a_C(e)=
\begin{cases}
|zC^{high}_e|,& \text{high pivots}\\
|zC^{low}_e|,& \text{low pivots}
\end{cases}
\]

显著性权重（threshold 模式）：
\[
w_{sig}(e)=SigPass_e
\]

若使用 bootstrap，可定义平滑权重：
\[
w_{sig}(e)=\left(1-\min(1,p^{price}_e/\alpha)\right)\cdot\left(1-\min(1,p^{cvd}_e/\alpha)\right)
\]

强度分数：
\[
score(e)=\operatorname{clip}_{[0,1]}\left(
w_{sig}(e)\cdot
\left[0.5\cdot \frac{a_P(e)}{z_p^{cap}} + 0.5\cdot \frac{a_C(e)}{z_c^{cap}}\right]
\right)
\]
其中 `\(z_p^{cap}, z_c^{cap}\)` 为饱和值（如 `3.0`），
\[
\operatorname{clip}_{[0,1]}(x)=\min(1,\max(0,x))
\]

### 3.12 双市场扩展（合约主判定 + 现货归因确认，修订版）
> **事件检测主逻辑保持不变**：3.8 的四类背离仍用 **合约价格 + 合约 CVD(F)**。  
> **新增归因**：使用 **现货 CVD(S)** 的去趋势标准化版本 \(Z^{C,S}\) 评估是否由现货主导。

对任一背离事件 \(e=(t_1,t_2)\)，定义方向化价格变化：
\[
\Delta P_e=
\begin{cases}
H[t_2]-H[t_1], & \text{若 } pivot\_side=\texttt{high}\\[4pt]
L[t_2]-L[t_1], & \text{若 } pivot\_side=\texttt{low}
\end{cases}
\]

定义 futures / spot 的 CVD 标准化差值（使用去趋势后的 robust-z）：
\[
\Delta Z^{C,F}_e = Z^{C,F}[t_2]-Z^{C,F}[t_1]
\]
\[
\Delta Z^{C,S}_e = Z^{C,S}[t_2]-Z^{C,S}[t_1]
\]

#### 3.12.1 现货是否支持价格方向（归因确认）
\[
SpotPriceFlowConfirm_e=
\mathbf{1}\{\operatorname{sign}(\Delta P_e)=\operatorname{sign}(\Delta Z^{C,S}_e)\}
\]

#### 3.12.2 合约侧是否背离（方向化符号）
\[
FutDivergenceSign_e=
\operatorname{sign}(\Delta P_e)\cdot \operatorname{sign}(\Delta Z^{C,F}_e)
\]
解释：
- `+1`：合约 CVD 与价格同向
- `-1`：合约 CVD 与价格背离
- `0`：中性（至少一侧近零）

#### 3.12.3 现货主导归因强度（修订）
\[
SpotLeadScore_e=
\frac{|\Delta Z^{C,S}_e|}{|\Delta Z^{C,S}_e|+|\Delta Z^{C,F}_e|+\varepsilon}
\cdot
\mathbf{1}\{\operatorname{sign}(\Delta P_e)=\operatorname{sign}(\Delta Z^{C,S}_e)\}
\cdot
\mathbf{1}\{\operatorname{sign}(\Delta P_e)\neq\operatorname{sign}(\Delta Z^{C,F}_e)\}
\]

> 相比旧版，直接使用 **已去趋势 + 标准化** 的 CVD 差值，避免不同市场量级差异导致分数偏移。

#### 3.12.4 可选标签（便于下游策略）
\[
likely\_driver(e)=
\begin{cases}
\texttt{spot\_led}, & SpotLeadScore_e\ge \theta_{spot}\\
\texttt{futures\_led}, & SpotLeadScore_e<\theta_{fut}\ \land\ FutDivergenceSign_e\neq -1\\
\texttt{mixed}, & \text{otherwise}
\end{cases}
\]

### 3.13 程序实现步骤（可直接落地）
1. **按 1m 聚合 F/S 逐笔成交**：得到 `Delta_1m^F`、`Delta_1m^S`。  
2. **构造 Rolling-7D CVD**：得到 `CVD_1m^{7d,F}`、`CVD_1m^{7d,S}`。  
3. **全因果去趋势**（只用过去数据）：
   - 价格：`log(C)` 做滚动 OLS/EMA 去趋势；
   - CVD(F/S)：各自做滚动 OLS/EMA 去趋势。  
4. **robust 标准化**：
   - 计算 `Z^P`, `Z^{C,F}`, `Z^{C,S}`（rolling median + MAD）。  
5. **检测 pivot（对称定义）并延迟确认**：
   - 记录 pivot 发生时刻 `pivot_ts`；
   - 记录确认时刻 `pivot_confirm_ts = pivot_ts + k*1m`；
   - 仅对“已确认 pivot”参与配对。  
6. **同侧相邻配对**（top 用 high pivots，bottom 用 low pivots）：
   - 约束 `G_min <= leg <= G_max`；
   - 过滤过短腿长（ATR 归一化）。  
7. **四类背离判定**（3.8）：
   - 价格用原始 `H/L`；
   - CVD 用 `Z^{C,F}`；
   - 加显著性 `SigPass`。  
8. **双市场归因**（3.12）：
   - 计算 `SpotPriceFlowConfirm`、`FutDivergenceSign`、`SpotLeadScore`。  
9. **输出事件**：
   - 必须带 `event_available_ts`（用于回测/线上一致性）；
   - 可同时输出 `pivot_ts` 与 `pivot_confirm_ts` 供可视化。  
10. **严格避免未来函数**：
   - 任何策略触发、标签、统计都使用 `event_available_ts` 之后的数据。

## 4) liquidation_density（合并多头/空头爆仓价格密集区域）

### 数据源要求（市场）
- **主计算市场**：合约（F）
- **现货数据（S）**：不需要（爆仓事件仅存在于合约市场）
- **必需数据类型**：合约 liquidation events（价格、名义金额、方向）

> 解决原文重复编号问题（两个 `19)`）。  
> 本容器统一输出：
> - `long` 爆仓密度（价格维度）
> - `short` 爆仓密度（价格维度）
> - `net_density = short - long`
> - `long_peak_zones / short_peak_zones`

### 输入
- liquidation events
  - `timestamp = \tau_j`
  - `price = p_j`
  - `notional = n_j`
  - `side ∈ {long_liq, short_liq}`
- `price_grid = \{x\}`
- 核函数 `K`
- 带宽 `h`
- `peak_prominence_threshold_long = \gamma_{long}`
- `peak_prominence_threshold_short = \gamma_{short}`
- `eps = 1e-12`

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）
- 对外输出分层：
  - current `1m` sidecar：当前分钟的 `levels / peak_levels / long_total / short_total`
  - `by_window["15m"|"1h"|"4h"|"1d"|"3d"]`：持仓后可见的窗口化爆仓密度视图
- 约束：current `1m` sidecar 不能作为唯一对外视图；持仓后判断必须能继续看到窗口级峰值区

### 计算公式

#### 4.1 多头爆仓密度（long liquidations）
\[
LongLiqDensity_w[t,x]=\sum_{j\in I_w(t),\,side_j=\text{long\_liq}} n_j\cdot K\!\left(\frac{x-p_j}{h}\right)
\]

#### 4.2 空头爆仓密度（short liquidations）
\[
ShortLiqDensity_w[t,x]=\sum_{j\in I_w(t),\,side_j=\text{short\_liq}} n_j\cdot K\!\left(\frac{x-p_j}{h}\right)
\]

#### 4.3 净爆仓密度（方向对比）
\[
NetLiqDensity_w[t,x]=ShortLiqDensity_w[t,x]-LongLiqDensity_w[t,x]
\]

对应 `levels[]` 字段：
- `long_liq_density`
- `short_liq_density`
- `net_liq_density`

#### 4.4 峰值区（Peak Zones，固定局部极值 + prominence）
定义平滑后密度（若 `K` 已平滑则可直接用原密度）：
\[
\widetilde L_w[t,x]\equiv LongLiqDensity_w[t,x],\qquad
\widetilde S_w[t,x]\equiv ShortLiqDensity_w[t,x]
\]

Long 峰值：
\[
x\in Peak^{long}_w[t]\iff
\widetilde L_w[t,x]>\widetilde L_w[t,x-\Delta p]\land
\widetilde L_w[t,x]>\widetilde L_w[t,x+\Delta p]\land
Prominence^{long}(x)\ge \gamma_{long}
\]

Short 峰值：
\[
x\in Peak^{short}_w[t]\iff
\widetilde S_w[t,x]>\widetilde S_w[t,x-\Delta p]\land
\widetilde S_w[t,x]>\widetilde S_w[t,x+\Delta p]\land
Prominence^{short}(x)\ge \gamma_{short}
\]

峰值评分（固定为同侧归一化）：
\[
score^{long}(x)=\frac{\widetilde L_w[t,x]}{\max_{y}\widetilde L_w[t,y]+\varepsilon}
\]
\[
score^{short}(x)=\frac{\widetilde S_w[t,x]}{\max_{y}\widetilde S_w[t,y]+\varepsilon}
\]

- 对外输出分层：
  - current `1m` sidecar：当前分钟 `levels[] / heatmap_summary / microprice_* / obi_* / ofi_*`
  - `by_window["15m"|"1h"]`：持仓后可见的窗口级 summary 视图
- 约束：
  - 顶层 `levels[]` 只代表 current minute，不代表窗口历史
  - 持仓后判断不能只依赖当前 `1m` snapshot，必须保留窗口级 `OBI / OFI / microprice / confirm / fake_order_risk` 语义

### 计算公式

#### 5.1 单时刻方向化深度（价格维度）
设在采样时刻 \(\tau\)、价位 \(x\) 的买/卖盘挂单量分别为：
\[
D^{bid}(\tau,x),\qquad D^{ask}(\tau,x)
\]

方向化总量与净量：
\[
H^{total}(\tau,x)=D^{bid}(\tau,x)+D^{ask}(\tau,x)
\]
\[
H^{net}(\tau,x)=D^{bid}(\tau,x)-D^{ask}(\tau,x)
\]

#### 5.2 窗口内方向化流动性热力图（时间平均）
对窗口 \(I_w(t)\) 内采样点 \(\tau_j\)（数量 \(N\)）：

买盘热力图：
\[
Heatmap^{bid}_w[t,x]=\frac{1}{N}\sum_{\tau_j\in I_w(t)} D^{bid}(\tau_j,x)
\]

卖盘热力图：
\[
Heatmap^{ask}_w[t,x]=\frac{1}{N}\sum_{\tau_j\in I_w(t)} D^{ask}(\tau_j,x)
\]

总流动性热力图：
\[
Heatmap^{total}_w[t,x]=Heatmap^{bid}_w[t,x]+Heatmap^{ask}_w[t,x]
\]

净流动性热力图：
\[
Heatmap^{net}_w[t,x]=Heatmap^{bid}_w[t,x]-Heatmap^{ask}_w[t,x]
\]

价位级失衡（便于热力图着色）：
\[
LevelImb_w[t,x]=\frac{Heatmap^{net}_w[t,x]}{Heatmap^{total}_w[t,x]+\varepsilon}
\]

对应 `levels[]` 字段：
- `bid_liquidity`
- `ask_liquidity`
- `total_liquidity`
- `net_liquidity`
- `level_imbalance`

#### 5.3 OBI（增强版：TWA + DW-OBI + ΔOBI + 流动性/价差调整；保留兼容字段）

##### 数据源要求（市场）
- **主计算**：合约盘口（F）
- **现货数据（S）**：需要（用于跨市场确认时并行计算 `OBI^S`）
- **必需数据类型（F/S）**：orderbook depth（前 K 档）或可重建前 K 档的 L2/L3 增量

> 在 `orderbook_depth_pack` 中，统一把 OBI 从“单一标量”升级为一组互补字段：  
> - `obi`（兼容旧字段，统一映射为 `obi_k_twa`）  
> - `obi_l1_twa` / `obi_k_twa`（时间加权）  
> - `obi_k_dw_twa`（距离加权）  
> - `obi_k_dw_change` / `obi_k_dw_slope`（动态变化）  
> - `obi_k_dw_adj_twa`（流动性/价差调整）  
> 同时保留 `spread_twa`、`topk_depth_twa` 便于解释。

**(1) Top-K 等权瞬时 OBI（保留旧定义）**
\[
OBI^{inst}_K[\tau]=
\frac{\sum_{k=1}^{K}Q^{bid}_k[\tau]-\sum_{k=1}^{K}Q^{ask}_k[\tau]}
{\sum_{k=1}^{K}Q^{bid}_k[\tau]+\sum_{k=1}^{K}Q^{ask}_k[\tau]+\varepsilon}
\]

**(2) 距离加权瞬时 OBI（DW-OBI，默认）**  
默认指数衰减权重（靠近盘口权重更高）：
\[
w_k=e^{-\lambda (k-1)},\qquad \lambda>0
\]
距离加权 OBI：
\[
OBI^{inst,dw}_K[\tau]=
\frac{\sum_{k=1}^{K}w_kQ^{bid}_k[\tau]-\sum_{k=1}^{K}w_kQ^{ask}_k[\tau]}
{\sum_{k=1}^{K}w_kQ^{bid}_k[\tau]+\sum_{k=1}^{K}w_kQ^{ask}_k[\tau]+\varepsilon}
\]

**(3) 窗口聚合固定使用时间加权均值（TWA）**  
设窗口区间 \(I_w(t)=(t-w,t]\)，对任意瞬时量 \(X(\tau)\)：
\[
X^{twa}_w[t]=\frac{1}{w}\int_{I_w(t)}X(\tau)\,d\tau
\]
离散/事件驱动实现使用阶梯函数（LOCF）精确积分，避免简单平均在不均匀采样下产生偏置。

因此：
\[
OBI^{twa}_{K,w}[t]=\frac{1}{w}\int_{I_w(t)}OBI^{inst}_K(\tau)\,d\tau
\]
\[
OBI^{twa,dw}_{K,w}[t]=\frac{1}{w}\int_{I_w(t)}OBI^{inst,dw}_K(\tau)\,d\tau
\]

字段映射统一：
- `obi_k = K`
- `obi = OBI^{twa}_{K,w}[t]`（兼容旧字段，语义升级）
- `obi_k_twa = OBI^{twa}_{K,w}[t]`
- `obi_l1_twa = OBI^{twa}_{1,w}[t]`
- `obi_k_dw_twa = OBI^{twa,dw}_{K,w}[t]`
- `obi_k_dw_close = OBI^{inst,dw}_K[t]`（窗口末快照值）

**(4) OBI 变化率 / 斜率（动态增强）**  
设固定滞后 \(\delta\)（如 1s / 5s）：
\[
\Delta OBI^{dw}_K[\tau]=OBI^{inst,dw}_K[\tau]-OBI^{inst,dw}_K[\tau-\delta]
\]
窗口时间加权均值：
\[
\Delta OBI^{twa,dw}_{K,w}[t]=\frac{1}{w}\int_{I_w(t)}\Delta OBI^{dw}_K(\tau)\,d\tau
\]
对应字段：
- `obi_k_dw_change = \Delta OBI^{twa,dw}_{K,w}[t]`
- `obi_k_dw_slope`（可选，窗口内线性回归斜率）

**(5) 流动性/价差调整版 OBI（固定）**  
定义 Top-K 总深度与价差：
\[
DepthTopK[\tau]=\sum_{k=1}^{K}Q^{bid}_k[\tau]+\sum_{k=1}^{K}Q^{ask}_k[\tau]
\]
\[
Spread[\tau]=p^{ask}_{1}[\tau]-p^{bid}_{1}[\tau]
\]
调整因子（固定可编码写法）：
\[
g[\tau]=
\operatorname{clip}_{[0,1]}\!\left(\frac{DepthTopK[\tau]}{depth_{ref}+\varepsilon}\right)
\cdot
\operatorname{clip}_{[0,1]}\!\left(\frac{spread_{ref}}{Spread[\tau]+\varepsilon}\right)
\]
其中 \(\operatorname{clip}_{[0,1]}(x)=\min(1,\max(0,x))\)。

调整版瞬时 OBI：
\[
OBI^{inst,dw,adj}_K[\tau]=OBI^{inst,dw}_K[\tau]\cdot g[\tau]
\]
窗口聚合：
\[
OBI^{twa,dw,adj}_{K,w}[t]=\frac{1}{w}\int_{I_w(t)}OBI^{inst,dw,adj}_K(\tau)\,d\tau
\]

统一新增字段：
- `spread_twa`
- `topk_depth_twa`
- `obi_k_dw_adj_twa`

**(6) 与 OFI 的协同（默认同 pack 输出，不替代 OBI）**  
- `OBI`：盘口存量倾斜（snapshot imbalance）  
- `OFI`：盘口变化倾斜（flow imbalance）  

若系统具备 BBO 事件流，可增补 `ofi_*` 字段（例如 `ofi_norm` / `ofi_ema`）作为 `orderbook_depth_pack` 的并行特征；用于预测时必须使用滞后特征，避免同桶信息泄漏。

### 5.4 Microprice（稳健版：幂次加权 + OBI 调整，防粉尘单/假挂单）

#### 数据源要求（市场）
- **主计算**：合约 BBO（F）
- **现货数据（S）**：不需要（本指标为单市场盘口微价格）
- **必需数据类型**：best bid/ask 价格与数量（BBO）

### 输入
- top1 最优盘口（每次采样 \(\tau\)）
  - `bid_price = p_b[\tau]`, `bid_qty = s_b[\tau]`
  - `ask_price = p_a[\tau]`, `ask_qty = s_a[\tau]`
- 可选：`OBI^{inst}_{N,\alpha}[\tau]`（来自 `5.3`）
- 参数
  - `kappa ∈ (0,1]`（默认统一 `0.35`）
  - `lambda_obi ∈ [0,0.5]`（默认统一 `0.15`）
  - `tick_size = Δp`
  - `size_floor`（统一最小数量地板，用于过滤粉尘）
  - `eps = 1e-12`

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）

### 计算公式

#### 5.4.1 经典 Microprice（保留对照）
\[
Microprice^{classic}[\tau]
=
\frac{s_b[\tau]\cdot p_a[\tau] + s_a[\tau]\cdot p_b[\tau]}
{s_b[\tau]+s_a[\tau]+\varepsilon}
\]

#### 5.4.2 幂次加权 Microprice（默认主字段）
先做数量地板与幂次压缩（抗粉尘/假墙）：
\[
\bar s_b=\max(s_b[\tau], size\_floor),\qquad
\bar s_a=\max(s_a[\tau], size\_floor)
\]
\[
\tilde s_b=(\bar s_b+\varepsilon)^\kappa,\qquad
\tilde s_a=(\bar s_a+\varepsilon)^\kappa
\]
\[
Microprice^{\kappa}[\tau]
=
\frac{\tilde s_b\cdot p_a[\tau] + \tilde s_a\cdot p_b[\tau]}
{\tilde s_b+\tilde s_a+\varepsilon}
\]

#### 5.4.3 OBI 调整的 Mid Price（可选增强）
\[
Mid[\tau]=\frac{p_a[\tau]+p_b[\tau]}{2}
\]
\[
Microprice^{adj}[\tau]
=
Mid[\tau]+\lambda_{obi}\cdot OBI^{inst}_{N,\alpha}[\tau]\cdot \frac{\Delta p}{2}
\]
并做区间截断（固定约束）：
\[
Microprice^{adj}[\tau]\in [p_b[\tau],\,p_a[\tau]]
\]

### 5.5 OFI（Order Flow Imbalance，Cont 风格 BBO 变化版）

#### 数据源要求（市场）
- **主计算**：合约 BBO 事件流（F）
- **现货数据（S）**：需要（用于并行 `OFI^S`，参与合约盘口真实性确认）
- **必需数据类型（F/S）**：BBO 连续更新事件流（best bid/ask price & qty）

### 输入
- BBO 事件流（逐次更新）
  - `timestamp_n`
  - `P^B_n, q^B_n`（best bid 价格/数量）
  - `P^A_n, q^A_n`（best ask 价格/数量）
- 聚合窗口 \(w\)（统一 `100ms` / `1s`）
- 标准化回看 `ofi_norm_lookback = N`
- `eps = 1e-12`

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）

### 计算公式

#### 5.5.1 顶层订单簿变化贡献（核心）
对 BBO 事件 \(n-1\to n\)，定义：
\[
e_n
=
\mathbf{1}\{P^B_n \ge P^B_{n-1}\}q^B_n
-
\mathbf{1}\{P^B_n \le P^B_{n-1}\}q^B_{n-1}
-
\mathbf{1}\{P^A_n \le P^A_{n-1}\}q^A_n
+
\mathbf{1}\{P^A_n \ge P^A_{n-1}\}q^A_{n-1}
\]

#### 5.5.2 窗口聚合 OFI
设窗口 \(I_w(t)\) 内 BBO 更新索引集合为 \(\mathcal{N}_w(t)\)：
\[
OFI_w[t]=\sum_{n\in \mathcal{N}_w(t)} e_n
\]

#### 5.5.3 标准化 OFI（固定）
\[
\mu^{ofi}_{w,N}[t]=\frac{1}{N}\sum_{k=1}^{N}OFI_w[t-k]
\]
\[
\sigma^{ofi}_{w,N}[t]=std(OFI_w[t-1],\dots,OFI_w[t-N])+\varepsilon
\]
\[
OFI^{norm}_{w,N}[t]=\frac{OFI_w[t]-\mu^{ofi}_{w,N}[t]}{\sigma^{ofi}_{w,N}[t]}
\]

> 边界约束：OFI 表示**限价单队列变化（BBO 结构变化）**；不等同于主动成交 `Trade Delta`，两者应并行输出。

### 5.6 Trades Burst（升级为方向化 CVD Burst：Delta / Relative Delta / EMA Burst）

#### 数据源要求（市场）
- **主计算**：合约逐笔成交（F）
- **现货数据（S）**：需要（用于并行 `Delta^S / CVD^S / Burst^S`，确认合约盘口真实性）
- **必需数据类型（F/S）**：逐笔成交（价格、数量、主动方向）

### 输入
- 逐笔成交 trades
  - `timestamp_i`, `price_i`, `qty_i`, `aggressor_side_i ∈ {+1,-1}`
- 聚合周期 \(b\)（统一 `100ms`）
- `burst_value_mode ∈ {qty, notional}`（默认统一 `qty`）
- EMA 参数：`ema_fast_len`, `ema_slow_len`
- 标准化回看 `burst_z_lookback = N`
- `eps = 1e-12`

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）

### 计算公式

#### 5.6.1 聚合到周期 \(b\) 的主动买/卖量
若 `burst_value_mode = qty`：
\[
V^{buy}_b[t]=\sum_{i\in I_b(t),\,a_i=+1} q_i,\qquad
V^{sell}_b[t]=\sum_{i\in I_b(t),\,a_i=-1} q_i
\]

若 `burst_value_mode = notional`：
\[
V^{buy}_b[t]=\sum_{i\in I_b(t),\,a_i=+1} p_i q_i,\qquad
V^{sell}_b[t]=\sum_{i\in I_b(t),\,a_i=-1} p_i q_i
\]

#### 5.6.2 Trade Delta（方向核心）
\[
\Delta_b[t]=V^{buy}_b[t]-V^{sell}_b[t]
\]

#### 5.6.3 Relative Delta（多空纯度）
\[
RelativeDelta_b[t]=
\frac{V^{buy}_b[t]-V^{sell}_b[t]}
{V^{buy}_b[t]+V^{sell}_b[t]+\varepsilon}
\]

#### 5.6.4 CVD（连续累积）
\[
CVD_b[t]=CVD_b[t-b]+\Delta_b[t]
\]

#### 5.6.5 CVD Burst（对 Delta 做快慢 EMA）
\[
\alpha_f=\frac{2}{ema\_fast\_len+1},\qquad
\alpha_s=\frac{2}{ema\_slow\_len+1}
\]
\[
EMA^{fast}_{\Delta}[t]=\alpha_f\Delta_b[t]+(1-\alpha_f)EMA^{fast}_{\Delta}[t-b]
\]
\[
EMA^{slow}_{\Delta}[t]=\alpha_s\Delta_b[t]+(1-\alpha_s)EMA^{slow}_{\Delta}[t-b]
\]
\[
CVDBurstSignal_b[t]=EMA^{fast}_{\Delta}[t]-EMA^{slow}_{\Delta}[t]
\]

#### 5.6.6 CVD Burst Z-score（固定）
\[
\mu^{burst}_N[t]=\frac{1}{N}\sum_{k=1}^{N}CVDBurstSignal_b[t-kb]
\]
\[
\sigma^{burst}_N[t]=std(CVDBurstSignal_b[t-b],\dots,CVDBurstSignal_b[t-Nb])+\varepsilon
\]
\[
CVDBurstZ_b[t]=\frac{CVDBurstSignal_b[t]-\mu^{burst}_N[t]}{\sigma^{burst}_N[t]}
\]

### 5.7 Continuous Absorption（连续化吸收：收益率-Delta 残差）

#### 数据源要求（市场）
- **主计算**：合约（F）
- **现货数据（S）**：不需要（本项为合约连续化微结构特征）
- **必需数据类型**：合约短周期 Delta、收益率、滚动波动率

### 输入
- `delta_1s[t]`（1s 主动成交差）
- `ret_1s[t]`（1s 收益率，统一 mid/last log return）
- `sigma_delta_1s[t]`, `sigma_ret_1s[t]`（滚动标准差）
- `beta_abs`（固定或滚动估计）
- `eps = 1e-12`

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）

### 计算公式
\[
Z^\Delta_{1s}[t]=\frac{\Delta_{1s}[t]}{\sigma^\Delta_{1s}[t]+\varepsilon}
\]
\[
Z^r_{1s}[t]=\frac{ret_{1s}[t]}{\sigma^r_{1s}[t]+\varepsilon}
\]
\[
AbsorptionIndex[t]=Z^\Delta_{1s}[t]-\beta_{abs}\cdot Z^r_{1s}[t]
\]

方向拆分（固定）：
\[
AbsorptionBidSide[t]=\max(AbsorptionIndex[t],0)
\]
\[
AbsorptionAskSide[t]=\max(-AbsorptionIndex[t],0)
\]

### 5.8 Continuous Footprint Imbalance（连续化足迹失衡 / 极端 Delta 密度）

#### 数据源要求（市场）
- **主计算**：合约（F）
- **现货数据（S）**：不需要（本项为合约连续化微结构特征）
- **必需数据类型**：合约短周期主动买卖量、趋势 z-score

### 输入
- `v_buy[t]`, `v_sell[t]`（统一 `100ms` 或 `1s`）
- `trend_z[t]`（慢尺度趋势 z-score）
- 参数
  - `imbalance_ratio_threshold = 3.0`
  - `trend_gate_z = 2.0`
  - `eps = 1`（离散防零）

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）

### 计算公式
\[
ImbalanceRatio[t]=
\max\left(
\frac{V_{buy}[t]}{V_{sell}[t]+\epsilon},
\frac{V_{sell}[t]}{V_{buy}[t]+\epsilon}
\right)
\]
\[
ImbSide[t]=
\begin{cases}
+1,& V_{buy}[t] > V_{sell}[t]\\
-1,& V_{sell}[t] > V_{buy}[t]\\
0,& \text{otherwise}
\end{cases}
\]
\[
ImbGate[t]=\mathbf{1}\{ImbalanceRatio[t] > imbalance\_ratio\_threshold \land |trend_z[t]| > trend\_gate\_z\}
\]
\[
InitiationGuard[t]=\mathbf{1}\{ImbGate[t]=1 \land sign(trend_z[t])=ImbSide[t]\}
\]

### 5.9 CVD Divergence Feature（连续化 CVD 背离因子）

#### 数据源要求（市场）
- **主计算**：合约（F）
- **现货数据（S）**：需要（输出 `CVDSlope^S`、构造 spot-driven divergence 归因）
- **必需数据类型（F/S）**：短周期 Trade Delta 序列（由逐笔成交聚合）、价格趋势 z-score（主趋势用合约）

### 输入
- `trade_delta_t`（统一 `100ms` 或 `1s`）
- `trend_z[t]`（价格慢尺度趋势 z-score）
- `N_div`（回看长度，统一如 `30s` 对应样本数）
- 标定常数 `z0`, `s0`
- `eps = 1e-12`

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）

### 计算公式

#### 5.9.1 CVD 构造
\[
CVD[t]=CVD[t-\Delta t]+\Delta_t
\]

#### 5.9.2 过去 \(N\) 样本 CVD 斜率（OLS）
在窗口 \(\{t-N+1,\dots,t\}\) 上，对 \((u, CVD[u])\) 做线性回归，取斜率：
\[
CVDSlope[t]=\hat\beta^{(cvd)}_t
\]

#### 5.9.3 背离因子（符号）
\[
DivergenceFactor[t]=sign(trend_z[t])\times sign(CVDSlope[t])
\]
- `+1`：趋势与 CVD 同向（延续）
- `-1`：趋势与 CVD 反向（背离）

#### 5.9.4 背离强度（连续值）
\[
DivergenceStrength[t]
=
\max\!\left(0,\frac{|trend_z[t]|}{z_0}\cdot\frac{|CVDSlope[t]|}{s_0+\varepsilon}\right)\cdot \mathbf{1}\{DivergenceFactor[t]=-1\}
\]

### 5.3 / 5.5 / 5.6 / 5.9 双市场修订（新增：现货确认 / 跨市场归因）

> 下列扩展不替代原公式；默认 **F（futures）为主市场**，**S（spot）为确认源**。  
> 统一统一窗口：`w ∈ {100ms, 1s}`（盘口/BBO/Trade Burst 连续特征一致化）。

#### 5.x.a 新增输入（统一）
- `orderbook_fut` / `bbo_fut`（合约）
- `orderbook_spot` / `bbo_spot`（现货）
- `trades_fut`, `trades_spot`（主动方向、ETH base qty 或 USDT notional）
- `K`（深度档位）、`w`（聚合窗口）
- 标准化回看：`N_obi`, `N_ofi`, `N_burst`, `N_div`
- （可选）L2/L3 增量统计：`add/cancel/exec`（用于假单增强版）

#### 5.x.b 双市场并行底层特征（适用于 5.3/5.5/5.6/5.9）
对 \(x\in\{F,S\}\) 分别计算（公式沿用原 5.3/5.5/5.6/5.9）：
- `OBI^{twa,dw,x}_{K,w}[t]`
- `OFI^x_w[t]`, `OFI^{norm,x}_{w,N}[t]`
- `\Delta^x_w[t]`（Trade Delta / Burst 同窗口聚合）
- `RelativeDelta^x_w[t]`
- `CVD^x[t]`
- `CVDSlope^x[t]`（5.9 OLS slope）

为便于跨市场比较，统一标准化：
\[
zOBI^x[t]=\frac{OBI^{twa,dw,x}_{K,w}[t]-\mu^{obi,x}[t]}{\sigma^{obi,x}[t]+\varepsilon}
\]
\[
zOFI^x[t]=\frac{OFI^x_w[t]-\mu^{ofi,x}[t]}{\sigma^{ofi,x}[t]+\varepsilon}
\]
\[
z\Delta^x[t]=\frac{\Delta^x_w[t]-\mu^{\Delta,x}[t]}{\sigma^{\Delta,x}[t]+\varepsilon}
\]

#### 5.x.c 合约盘口信号是否被成交验证（合约内确认）
\[
ExecConfirm^F[t]=
\mathbf{1}\left\{
\operatorname{sign}(OBI^{twa,dw,F}_{K,w}[t])=
\operatorname{sign}(OFI^{F}_{w}[t])=
\operatorname{sign}(\Delta^{F}_{w}[t])
\right\}
\]

#### 5.x.d 现货是否同向确认（跨市场确认）
\[
SpotConfirm^F[t]=
\mathbf{1}\left\{
\operatorname{sign}(OBI^{twa,dw,F}_{K,w}[t])=\operatorname{sign}(\Delta^{S}_{w}[t])
\ \lor\
\operatorname{sign}(OBI^{twa,dw,F}_{K,w}[t])=\operatorname{sign}(OFI^{S}_{w}[t])
\right\}
\]

> 默认新增输出字段（窗口级）：
> - `exec_confirm_fut`
> - `spot_confirm`
> - `obi_fut`, `obi_spot`, `ofi_fut`, `ofi_spot`, `trade_delta_fut`, `trade_delta_spot`

#### 5.x.e 疑似虚假订单风险（无 L3 版本，概率型）
定义合约盘口冲击强度（用 OBI shock 表示）：
\[
OBIShock^F[t]=\frac{|OBI^{twa,dw,F}_{K,w}[t]-\mu^{obi,F}[t]|}{\sigma^{obi,F}[t]+\varepsilon}
\]
定义同窗价格响应弱（合约）：
\[
WeakPriceResp^F[t]=\mathbf{1}\{|r^F_w[t]|<r_{min}\}
\]
则疑似假单风险：
\[
FakeOrderRisk^F[t]
=
\operatorname{clip}_{[0,1]}\!\left(\frac{OBIShock^F[t]}{z_{ref}}\right)
\cdot (1-ExecConfirm^F[t])
\cdot (1-SpotConfirm^F[t])
\cdot WeakPriceResp^F[t]
\]

统一新增字段：
- `fake_order_risk_fut`
- `obi_shock_fut`
- `weak_price_resp_fut`

#### 5.x.f 若可用 L2/L3 增量：假单增强版（固定）
对合约侧 Top-K（窗口 \(w\)）：
- `Add^{side,F}_{K,w}[t]`
- `Cancel^{side,F}_{K,w}[t]`
- `Exec^{side,F}_{K,w}[t]`

撤单压力：
\[
CancelPressure^{ask,F}_{K,w}[t]=
\frac{Cancel^{ask,F}_{K,w}[t]}{Add^{ask,F}_{K,w}[t]+Exec^{ask,F}_{K,w}[t]+\varepsilon}
\]
\[
CancelPressure^{bid,F}_{K,w}[t]=
\frac{Cancel^{bid,F}_{K,w}[t]}{Add^{bid,F}_{K,w}[t]+Exec^{bid,F}_{K,w}[t]+\varepsilon}
\]

可定义增强风险（示意）：
\[
FakeOrderRisk^{F,L3}[t]=FakeOrderRisk^F[t]\cdot
\max\big(CancelPressure^{ask,F}_{K,w}[t],\ CancelPressure^{bid,F}_{K,w}[t]\big)
\]

#### 5.x.g 连续化 CVD 背离（5.9）双市场归因扩展
保留原定义：
\[
DivergenceFactor^F[t]=\operatorname{sign}(trend_z[t])\cdot \operatorname{sign}(CVDSlope^F[t])
\]
新增现货侧同口径：
\[
DivergenceFactor^S[t]=\operatorname{sign}(trend_z[t])\cdot \operatorname{sign}(CVDSlope^S[t])
\]

现货驱动背离标记（合约背离、现货确认）：
\[
SpotDrivenDivergenceFlag[t]
=
\mathbf{1}\{DivergenceFactor^F[t]=-1 \land DivergenceFactor^S[t]=+1\}
\]

定义斜率标准化：
\[
zSlope^x[t]=\frac{CVDSlope^x[t]}{\sigma^{slope,x}[t]+\varepsilon}
\]
跨市场 CVD 归因强度：
\[
CrossCVDAttribution[t]=
\frac{zSlope^S[t]-zSlope^F[t]}{|zSlope^S[t]|+|zSlope^F[t]|+\varepsilon}
\]

统一新增字段（5.9）：
- `cvd_slope_fut`, `cvd_slope_spot`
- `divergence_factor_fut`, `divergence_factor_spot`
- `spot_driven_divergence_flag`
- `cross_cvd_attribution`

## 6) absorption（吸收，方向事件；分钟级底层计算；窗口内多事件）

### 数据源要求（市场）
- **主判定市场**：合约（F）
- **现货数据（S）**：**需要**（用于 spot flow confirm / spot whale confirm，提升吸收识别精度）
- **必需数据类型（F）**：合约 1m OHLCV、合约逐笔成交、合约关键位（VA/POC/AVWAP 等）
- **必需数据类型（S）**：现货逐笔成交（1m Delta / RelativeDelta / CVD），可选现货大额交易统计

> **双市场修订（合约主判定 + 现货 flow confirm）**
> - 吸收事件主判定仍按合约分钟级结构完成
> - 新增 `spot flow confirm / spot whale confirm`，减少“合约单边吸收假象”

### 输入
- **分钟级 OHLCV（1m）**
  - `ts_1m[m], O[m], H[m], L[m], C[m], Vol[m]`
- **逐笔成交 trades**（用于 1m footprint / delta）
  - `timestamp_i, price_i, qty_i, aggressor_side_i∈{+1,-1}`
- `tick_size = Δp`
- **分钟级 stacked imbalance**
  - `StackedBuy_1m[m] ∈ {0,1}`
  - `StackedSell_1m[m] ∈ {0,1}`
- **分钟级关键位（前向填充）**
  - `VAH[m], VAL[m], POC[m], AVWAP[m], PrevSessionHigh[m], PrevSessionLow[m]`
- 参数（固定默认值，可直接编码）
  - `eta_reject = 0.70`
  - `min_range_ticks = 4`
  - `rdelta_abs_min = 0.15`
  - `key_dist_ticks = 6`
  - `merge_gap_minutes = 1`
  - `confirm_bars = 3`
  - `max_event_minutes = 30`
  - `eps = 1e-12`

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）

### 计算公式

#### 6.1 分钟级 Delta / Relative Delta（1m）
\[
\Delta_{1m}[m]=\sum_{i\in I_{1m}(m)} a_i q_i,\qquad a_i\in\{+1,-1\}
\]
\[
RD_{1m}[m]=\frac{\Delta_{1m}[m]}{Vol[m]+\varepsilon}
\]

#### 6.2 分钟级关键位距离（用于“吸收发生在结构位附近”）
定义关键位集合：
\[
\mathcal{K}[m]=\{VAH[m],VAL[m],POC[m],AVWAP[m],PrevSessionHigh[m],PrevSessionLow[m]\}
\]

底部吸收用低点到关键位最小距离：
\[
d^{low}_{key}[m]=\min_{k\in\mathcal{K}[m]} \frac{|L[m]-k|}{\Delta p}
\]

顶部吸收用高点到关键位最小距离：
\[
d^{high}_{key}[m]=\min_{k\in\mathcal{K}[m]} \frac{|H[m]-k|}{\Delta p}
\]

#### 6.3 分钟级拒绝强度（K线收盘回拉/回落）
先定义分钟波动范围：
\[
R[m]=H[m]-L[m]
\]

看涨吸收的底部拒绝比：
\[
RejectBull[m]=\frac{C[m]-L[m]}{R[m]+\varepsilon}
\]

看跌吸收的顶部拒绝比：
\[
RejectBear[m]=\frac{H[m]-C[m]}{R[m]+\varepsilon}
\]

#### 6.4 单分钟吸收候选（严格版，无“可选项”）
**看涨吸收候选（吸收卖压）**：
\[
CandAbs^{bull}[m]=\mathbf{1}\Big(
StackedSell_{1m}[m]=1\ \land\
R[m]\ge min\_range\_ticks\cdot \Delta p\ \land\
RD_{1m}[m]\le -rdelta\_abs\_min\ \land\
C[m]>O[m]\ \land\
RejectBull[m]\ge \eta_{reject}\ \land\
d^{low}_{key}[m]\le key\_dist\_ticks
\Big)
\]

**看跌吸收候选（吸收买压）**：
\[
CandAbs^{bear}[m]=\mathbf{1}\Big(
StackedBuy_{1m}[m]=1\ \land\
R[m]\ge min\_range\_ticks\cdot \Delta p\ \land\
RD_{1m}[m]\ge rdelta\_abs\_min\ \land\
C[m]<O[m]\ \land\
RejectBear[m]\ge \eta_{reject}\ \land\
d^{high}_{key}[m]\le key\_dist\_ticks
\Big)
\]

#### 6.5 事件合并（同方向候选分钟合并）
定义方向标签：
\[
s[m]=
\begin{cases}
+1,& CandAbs^{bull}[m]=1 \land CandAbs^{bear}[m]=0\\
-1,& CandAbs^{bear}[m]=1 \land CandAbs^{bull}[m]=0\\
0,& \text{otherwise}
\end{cases}
\]

将满足以下条件的分钟索引合并为同一事件段 \(G=(m_s,\dots,m_e)\)：
- \(s[m]\) 同号且非零；
- 相邻有效分钟间隔 \(\le merge\_gap\_minutes\)；
- 持续时长 \((m_e-m_s+1)\le max\_event\_minutes\)。

#### 6.6 事件确认与结束时间（必须确认，不确认不输出）
对候选事件段 \(G=[m_s,m_e]\)：

段内极值与范围：
- 若为看涨吸收（\(s=+1\)）：
\[
L_G=\min_{m\in G}L[m],\quad H_G=\max_{m\in G}H[m],\quad R_G=H_G-L_G
\]
确认条件（未来 `confirm_bars` 分钟内）：
\[
\exists j\in\{1,\dots,confirm\_bars\}:\ C[m_e+j]\ge L_G+0.5R_G
\]
且
\[
\min_{j=1,\dots,confirm\_bars}L[m_e+j]\ge L_G-\Delta p
\]

- 若为看跌吸收（\(s=-1\)）：
\[
L_G=\min_{m\in G}L[m],\quad H_G=\max_{m\in G}H[m],\quad R_G=H_G-L_G
\]
确认条件：
\[
\exists j\in\{1,\dots,confirm\_bars\}:\ C[m_e+j]\le H_G-0.5R_G
\]
且
\[
\max_{j=1,\dots,confirm\_bars}H[m_e+j]\le H_G+\Delta p
\]

令最早满足条件的分钟为 \(m_c\)。若不存在 \(m_c\)，该候选段丢弃。  
最终事件时间范围：
\[
start\_ts=t(m_s),\quad end\_ts=t(m_e),\quad confirm\_ts=t(m_c)
\]

#### 6.7 事件字段与强度分数（固定公式）
设事件段 \(G\) 的分钟集合为 \(\{m_s,\dots,m_e\}\)，长度 \(N_G=m_e-m_s+1\)。

事件方向：
- \(s=+1 \Rightarrow type=\texttt{bullish\_absorption}\)
- \(s=-1 \Rightarrow type=\texttt{bearish\_absorption}\)

段内统计量：
\[
\Delta^{sum}_G=\sum_{m\in G}\Delta_{1m}[m],\qquad
RD^{mean}_G=\frac{1}{N_G}\sum_{m\in G}RD_{1m}[m]
\]
\[
Reject_G=
\begin{cases}
\frac{1}{N_G}\sum_{m\in G}RejectBull[m],& s=+1\\[4pt]
\frac{1}{N_G}\sum_{m\in G}RejectBear[m],& s=-1
\end{cases}
\]
\[
d^{key}_G=
\begin{cases}
\min_{m\in G} d^{low}_{key}[m],& s=+1\\
\min_{m\in G} d^{high}_{key}[m],& s=-1
\end{cases}
\]

定义归一化函数 \(\operatorname{clip}_{[0,1]}(x)=\min(1,\max(0,x))\)。  
强度分数：
\[
score_G=
0.35\cdot \operatorname{clip}_{[0,1]}\!\left(\frac{|RD^{mean}_G|}{0.5}\right)
+0.35\cdot \operatorname{clip}_{[0,1]}\!\left(\frac{Reject_G-\eta_{reject}}{1-\eta_{reject}}\right)
+0.20\cdot \operatorname{clip}_{[0,1]}\!\left(1-\frac{d^{key}_G}{key\_dist\_ticks}\right)
+0.10\cdot \operatorname{clip}_{[0,1]}\!\left(\frac{N_G}{5}\right)
\]

`pivot_price` 定义为：
- 看涨吸收：\(\min_{m\in G} L[m]\)
- 看跌吸收：\(\max_{m\in G} H[m]\)

#### 6.8 双市场扩展（新增：spot flow confirm / whale confirm）

**新增输入（覆盖补充）**
- `trades_spot`（ETHUSDT 现货逐笔成交，主动方向）
- （可选）`whale_trades_spot_1m[m]` 或 `WhaleDeltaNotional^S_{1m}[m]`（来自 `15)`）
- `spot_confirm_theta_rd = \theta_{rd}`（默认统一 `0.10~0.20`）
- `spot_confirm_theta_whale = \theta_{whale}`（按名义金额标定）

对现货侧分钟级特征（固定 1m）：
\[
\Delta^{S}_{1m}[m]=\sum_{i\in I^{S}_{1m}(m)} a_i q_i^{(S,ETH)}
\]
\[
RD^{S}_{1m}[m]=\frac{\Delta^{S}_{1m}[m]}{Vol^{S}[m]+\varepsilon}
\]
\[
CVD^{S}_{1m}[m]=CVD^{S}_{1m}[m-1]+\Delta^{S}_{1m}[m]
\]

对事件段 \(G=[m_s,m_e]\)，令方向
\[
s=
\begin{cases}
+1,& \text{bullish\_absorption}\\
-1,& \text{bearish\_absorption}
\end{cases}
\]

定义现货流确认分数（方向一致则加分）：
\[
SpotFlowConfirm_G=
\operatorname{clip}_{[0,1]}
\left(
\frac{s\cdot \frac{1}{N_G}\sum_{m\in G} RD^{S}_{1m}[m]}{\theta_{rd}+\varepsilon}
\right)
\]

若接入现货大额交易（固定）：
\[
SpotWhaleConfirm_G=
\operatorname{clip}_{[0,1]}
\left(
\frac{s\cdot \sum_{m\in G} WhaleDeltaNotional^S_{1m}[m]}{\theta_{whale}+\varepsilon}
\right)
\]

新增字段：
- `spot_rdelta_1m_mean`
- `spot_cvd_1m_change`
- `spot_flow_confirm`
- `spot_whale_confirm`（可选）

**强度分数修订（默认，不破坏原主判定）**
\[
score'_G
=
0.85\cdot score_G
+0.10\cdot SpotFlowConfirm_G
+0.05\cdot SpotWhaleConfirm_G
\]
若未接入现货大额交易，可退化为：
\[
score'_G=0.90\cdot score_G+0.10\cdot SpotFlowConfirm_G
\]

## 7) initiation（发起，方向事件；分钟级底层计算；窗口内多事件）

### 数据源要求（市场）
- **主判定市场**：合约（F）
- **现货数据（S）**：**需要**（用于 breakout confirmation，过滤假突破）
- **必需数据类型（F）**：合约 1m OHLCV、合约逐笔成交、合约关键位/结构特征
- **必需数据类型（S）**：现货逐笔成交（1m Delta / RelativeDelta / CVD），可选现货大额交易统计

> **双市场修订（合约主判定 + 现货确认过滤假突破）**
> - 发起/突破事件主条件不变（合约）
> - 新增现货 `RD/CVD/Whale` 确认项，提高突破真实性

### 输入
- **分钟级 OHLCV（1m）**
- **逐笔成交 trades**（用于 1m Delta / footprint）
- **分钟级 stacked imbalance**
  - `StackedBuy_1m[m]`, `StackedSell_1m[m]`
- **分钟级关键位**
  - `VAH[m]`, `VAL[m]`
- 参数（固定默认值）
  - `epsilon_break_ticks = 2`
  - `zdelta_lookback = 60`
  - `zdelta_min = 1.5`
  - `rdelta_min = 0.20`
  - `min_follow_minutes = 5`
  - `hold_break_ticks = 1`
  - `max_event_minutes = 60`
  - `eps = 1e-12`

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）

### 计算公式

#### 7.1 分钟级 Delta / Relative Delta（1m）
\[
\Delta_{1m}[m]=\sum_{i\in I_{1m}(m)} a_i q_i,\qquad
RD_{1m}[m]=\frac{\Delta_{1m}[m]}{Vol[m]+\varepsilon}
\]

#### 7.2 Delta z-score（滚动标准化，固定用标准差）
\[
\mu_{\Delta}[m]=\frac{1}{n}\sum_{j=0}^{n-1}\Delta_{1m}[m-j],\quad n=zdelta\_lookback
\]
\[
\sigma_{\Delta}[m]=std(\Delta_{1m}[m-n+1],\dots,\Delta_{1m}[m])+\varepsilon
\]
\[
Z_{\Delta}[m]=\frac{\Delta_{1m}[m]-\mu_{\Delta}[m]}{\sigma_{\Delta}[m]}
\]

#### 7.3 单分钟发起候选（必须同时满足突破 + 订单流强度 + stacked）
定义突破价格阈值：
\[
\epsilon_{break}=epsilon\_break\_ticks\cdot \Delta p,\qquad
\epsilon_{hold}=hold\_break\_ticks\cdot \Delta p
\]

K线收盘位置比（Close Location Value）：
\[
CLV[m]=\frac{C[m]-L[m]}{H[m]-L[m]+\varepsilon}
\]
\[
CLV^{bear}[m]=\frac{H[m]-C[m]}{H[m]-L[m]+\varepsilon}
\]

**看涨发起候选**：
\[
CandInit^{bull}[m]=\mathbf{1}\Big(
C[m] > VAH[m]+\epsilon_{break}
\land C[m-1]\le VAH[m-1]+\epsilon_{break}
\land Z_{\Delta}[m]\ge zdelta\_min
\land RD_{1m}[m]\ge rdelta\_min
\land StackedBuy_{1m}[m]=1
\land CLV[m]\ge 0.70
\Big)
\]

**看跌发起候选**：
\[
CandInit^{bear}[m]=\mathbf{1}\Big(
C[m] < VAL[m]-\epsilon_{break}
\land C[m-1]\ge VAL[m-1]-\epsilon_{break}
\land Z_{\Delta}[m]\le -zdelta\_min
\land RD_{1m}[m]\le -rdelta\_min
\land StackedSell_{1m}[m]=1
\land CLV^{bear}[m]\ge 0.70
\Big)
\]

#### 7.4 延续确认（必须满足，避免假突破）
对于候选起点 \(m_0\)：

**看涨发起确认**（未来 `min_follow_minutes` 分钟内持续接受在 VAH 上方）：
\[
Follow^{bull}(m_0)=\mathbf{1}\Big(
\min_{j=1,\dots,min\_follow\_minutes} C[m_0+j]\ge VAH[m_0]-\epsilon_{hold}
\land
\sum_{j=0}^{min\_follow\_minutes}\Delta_{1m}[m_0+j] > 0
\Big)
\]

**看跌发起确认**：
\[
Follow^{bear}(m_0)=\mathbf{1}\Big(
\max_{j=1,\dots,min\_follow\_minutes} C[m_0+j]\le VAL[m_0]+\epsilon_{hold}
\land
\sum_{j=0}^{min\_follow\_minutes}\Delta_{1m}[m_0+j] < 0
\Big)
\]

若确认不成立，则该候选不输出事件。

#### 7.5 事件结束时间（固定规则）
看涨发起事件从候选分钟开始，直到首次失守突破位或达到最大时长：
\[
m_e^{bull}=\min\Big\{m\ge m_0+min\_follow\_minutes\ \big|\ C[m] < VAH[m_0]-\epsilon_{hold}\Big\}-1
\]
若不存在，则
\[
m_e^{bull}=m_0+max\_event\_minutes-1
\]

看跌发起：
\[
m_e^{bear}=\min\Big\{m\ge m_0+min\_follow\_minutes\ \big|\ C[m] > VAL[m_0]+\epsilon_{hold}\Big\}-1
\]
若不存在则同理截断到 `max_event_minutes`。

确认时间固定定义：
\[
confirm\_ts = t(m_0+min\_follow\_minutes)
\]

#### 7.6 事件强度分数（固定公式）
事件段 \(G=[m_0,m_e]\)，长度 \(N_G\)：
\[
\Delta^{sum}_G=\sum_{m\in G}\Delta_{1m}[m],\qquad RD^{mean}_G=\frac{1}{N_G}\sum_{m\in G}RD_{1m}[m]
\]

突破幅度（看涨）：
\[
BreakMag^{bull}=\frac{C[m_0]-VAH[m_0]}{\Delta p}
\]
（看跌则替换为 \((VAL[m_0]-C[m_0])/\Delta p\)）

强度分数：
\[
score_G=
0.30\cdot \operatorname{clip}_{[0,1]}\!\left(\frac{|Z_{\Delta}[m_0]|}{3}\right)
+0.30\cdot \operatorname{clip}_{[0,1]}\!\left(\frac{|RD^{mean}_G|}{0.5}\right)
+0.20\cdot \operatorname{clip}_{[0,1]}\!\left(\frac{BreakMag}{6}\right)
+0.20\cdot \operatorname{clip}_{[0,1]}\!\left(\frac{N_G}{max\_event\_minutes}\right)
\]

其中 \(\operatorname{clip}_{[0,1]}\) 同上。

#### 7.7 双市场扩展（新增：spot breakout confirmation）

**新增输入（覆盖补充）**
- `trades_spot`（ETHUSDT 现货逐笔成交）
- （可选）`WhaleDeltaNotional^S_{1m}[m]`（来自 `15)`）
- 参数
  - `spot_rdelta_min = 0.05`
  - `spot_cvd_confirm_min = 0`
  - `spot_break_confirm_mode ∈ {score_only, strict_gate}`（默认 `score_only`）

现货分钟级特征：
\[
\Delta^{S}_{1m}[m]=\sum_{i\in I^{S}_{1m}(m)} a_i q_i^{(S,ETH)},\qquad
RD^{S}_{1m}[m]=\frac{\Delta^{S}_{1m}[m]}{Vol^{S}[m]+\varepsilon}
\]
\[
CVD^{S}_{1m}[m]=CVD^{S}_{1m}[m-1]+\Delta^{S}_{1m}[m]
\]

对候选起点 \(m_0\)，令方向
\[
s=
\begin{cases}
+1,& \text{bullish\_initiation}\\
-1,& \text{bearish\_initiation}
\end{cases}
\]

定义发起确认窗口：
\[
\mathcal{W}_{follow}(m_0)=\{m_0,\dots,m_0+min\_follow\_minutes\}
\]

现货相对 Delta 确认：
\[
SpotRDConfirm(m_0)=
\mathbf{1}\left\{
s\cdot \frac{1}{|\mathcal{W}_{follow}|}\sum_{m\in \mathcal{W}_{follow}(m_0)}RD^{S}_{1m}[m]
\ge spot\_rdelta\_min
\right\}
\]

现货 CVD 确认：
\[
SpotCVDConfirm(m_0)=
\mathbf{1}\left\{
s\cdot \big(CVD^{S}_{1m}[m_0+min\_follow\_minutes]-CVD^{S}_{1m}[m_0]\big)
\ge spot\_cvd\_confirm\_min
\right\}
\]

综合现货突破确认：
\[
SpotBreakConfirm(m_0)=\mathbf{1}\{SpotRDConfirm(m_0)=1 \land SpotCVDConfirm(m_0)=1\}
\]

若接入现货大额交易，定义：
\[
SpotWhaleBreakConfirm(m_0)=
\mathbf{1}\left\{
s\cdot \sum_{m\in \mathcal{W}_{follow}(m_0)}WhaleDeltaNotional^S_{1m}[m] > 0
\right\}
\]

> **可选严格过滤（仅当 `spot_break_confirm_mode = strict_gate`）**
> - 在 `7.4 延续确认` 的基础上额外要求 `SpotBreakConfirm(m_0)=1`
> - 默认统一 **不作为硬门槛**，而是进入强度分数加权（避免漏掉合约主导突破）

**强度分数修订（固定）**
对已确认事件段 \(G=[m_0,m_e]\)：
\[
SpotFlowConfirm_G=
\operatorname{clip}_{[0,1]}
\left(
s\cdot \frac{1}{N_G}\sum_{m\in G}RD^{S}_{1m}[m]
\right)
\]
\[
SpotWhaleConfirm_G=\mathbf{1}\left\{s\cdot \sum_{m\in G}WhaleDeltaNotional^S_{1m}[m] > 0\right\}
\]
\[
score'_G
=
0.80\cdot score_G
+0.15\cdot SpotFlowConfirm_G
+0.05\cdot SpotWhaleConfirm_G
\]

统一新增字段：
- `spot_break_confirm`
- `spot_rdelta_1m_mean`
- `spot_cvd_change`
- `spot_whale_break_confirm`（可选）
- `strength_score_xmk`（双市场修订分数）

## 12) buying exhaustion（买入衰竭，事件型，多事件，分钟级底层计算）

### 数据源要求（市场）
- **主判定市场**：合约（F）
- **现货数据（S）**：**需要**（用于区分真衰竭 vs 现货扫盘延续）
- **必需数据类型（F）**：合约 1m OHLCV、合约逐笔成交、合约事件判定所需结构特征
- **必需数据类型（S）**：现货逐笔成交（spot CVD / spot Delta），现货大额交易统计（Whale）

> **双市场修订（真衰竭 vs 现货扫盘延续）**
> - 买入衰竭主判定仍按合约侧（价格高点 + Delta/RDelta 弱化 + 拒绝）
> - 新增现货 `CVD + Whale` 确认/否决字段，用于区分“真衰竭”与“现货继续推升”

### 输入
- **分钟级 OHLCV（1m）**
- **逐笔成交 trades**（用于 1m Delta）
- 参数（固定默认值）
  - `pivot_left = 3`
  - `pivot_right = 3`
  - `min_leg_gap_minutes = 3`
  - `max_leg_gap_minutes = 180`
  - `epsilon_price_ticks = 2`
  - `epsilon_delta = 20`（按标的单位自行标定）
  - `epsilon_rdelta = 0.05`
  - `eta_reject = 0.65`
  - `confirm_bars = 5`
  - `epsilon_confirm_ticks = 2`
  - `eps = 1e-12`

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）

### 计算公式

#### 12.1 分钟级 Delta / Relative Delta / CVD（1m）
\[
\Delta_{1m}[m]=\sum_{i\in I_{1m}(m)} a_i q_i
\]
\[
RD_{1m}[m]=\frac{\Delta_{1m}[m]}{Vol[m]+\varepsilon}
\]
\[
CVD_{1m}[m]=CVD_{1m}[m-1]+\Delta_{1m}[m]
\]

#### 12.2 分钟级 swing high（严格 pivot）
令 \(k_l=pivot\_left,\ k_r=pivot\_right\)。定义：
\[
SwingHigh[m]=\mathbf{1}\Big(
H[m]>\max(H[m-k_l],\dots,H[m-1])
\ \land\
H[m]\ge \max(H[m+1],\dots,H[m+k_r])
\Big)
\]
高点索引集合：
\[
\mathcal{P}^{high}=\{m\mid SwingHigh[m]=1\}
\]

#### 12.3 顶部拒绝比（高点分钟）
\[
R[m]=H[m]-L[m],\qquad RejectTop[m]=\frac{H[m]-C[m]}{R[m]+\varepsilon}
\]

#### 12.4 相邻高点买入衰竭事件判定（固定规则）
对 \(\mathcal{P}^{high}\) 中相邻高点 \(t_1<t_2\)，仅当：
\[
min\_leg\_gap\_minutes \le t_2-t_1 \le max\_leg\_gap\_minutes
\]
才参与判定。

定义阈值：
\[
\epsilon_p = epsilon\_price\_ticks\cdot \Delta p,\qquad
\epsilon_c = epsilon\_confirm\_ticks\cdot \Delta p
\]

买入衰竭成立当且仅当同时满足：
\[
H[t_2]\ge H[t_1]+\epsilon_p
\]
\[
\Delta_{1m}[t_2]\le \Delta_{1m}[t_1]-\epsilon_{delta}
\]
\[
RD_{1m}[t_2]\le RD_{1m}[t_1]-\epsilon_{rdelta}
\]
\[
RejectTop[t_2]\ge \eta_{reject}
\]

并且在未来 `confirm_bars` 分钟内出现下行确认（首次确认分钟 \(t_c\)）：
\[
t_c=\min\Big\{t\in \{t_2+1,\dots,t_2+confirm\_bars\}\ \big|\ C[t]\le L[t_2]-\epsilon_c \Big\}
\]
若上述集合为空，则不输出该事件。

事件定义：
\[
e=(t_1,t_2,t_c)
\]
时间范围：
\[
start\_ts=t(t_1),\quad end\_ts=t(t_c),\quad confirm\_ts=t(t_c)
\]

#### 12.5 事件强度分数（固定公式）
\[
PricePush=\frac{H[t_2]-H[t_1]}{\Delta p},\quad
DeltaDrop=\Delta_{1m}[t_1]-\Delta_{1m}[t_2],\quad
RDeltaDrop=RD_{1m}[t_1]-RD_{1m}[t_2]
\]
\[
ConfirmSpeed=\frac{1}{t_c-t_2}
\]
\[
score=
0.30\cdot \operatorname{clip}_{[0,1]}\!\left(\frac{PricePush}{10}\right)
+0.25\cdot \operatorname{clip}_{[0,1]}\!\left(\frac{DeltaDrop}{3\epsilon_{delta}}\right)
+0.20\cdot \operatorname{clip}_{[0,1]}\!\left(\frac{RDeltaDrop}{3\epsilon_{rdelta}}\right)
+0.15\cdot \operatorname{clip}_{[0,1]}\!\left(\frac{RejectTop[t_2]-\eta_{reject}}{1-\eta_{reject}}\right)
+0.10\cdot \operatorname{clip}_{[0,1]}\!\left(ConfirmSpeed\cdot confirm\_bars\right)
\]

#### 12.6 双市场扩展（新增：spot CVD / spot whale 归因）

**新增输入（覆盖补充）**
- `trades_spot`（ETHUSDT 现货逐笔成交）
- `WhaleDeltaNotional^S_{1m}[m]`（可选，来自 `15)`）
- 参数
  - `spot_cvd_cont_threshold = \theta^{buy}_{cvd}`
  - `spot_whale_cont_threshold = \theta^{buy}_{whale}`
  - `lambda_spot_penalty = \lambda_s`（默认 `0.15~0.30`）

现货分钟级特征：
\[
\Delta^S_{1m}[m]=\sum_{i\in I^S_{1m}(m)} a_i q_i^{(S,ETH)},\qquad
RD^S_{1m}[m]=\frac{\Delta^S_{1m}[m]}{Vol^S[m]+\varepsilon}
\]
\[
CVD^S_{1m}[m]=CVD^S_{1m}[m-1]+\Delta^S_{1m}[m]
\]

对买入衰竭事件 \(e=(t_1,t_2,t_c)\)，定义现货在高点后的延续强度：
\[
SpotCVDPush^{buy}_e = CVD^S_{1m}[t_c]-CVD^S_{1m}[t_2]
\]
\[
SpotWhalePush^{buy}_e = \sum_{m=t_2}^{t_c} WhaleDeltaNotional^S_{1m}[m]
\]

**(a) 现货延续风险（说明“可能并非真衰竭”）**
\[
SpotContinuationRisk^{buy}_e=
\mathbf{1}\{SpotCVDPush^{buy}_e>\theta^{buy}_{cvd}\}
\ \lor\
\mathbf{1}\{SpotWhalePush^{buy}_e>\theta^{buy}_{whale}\}
\]

**(b) 现货衰竭确认（说明“更像真衰竭”）**
\[
SpotExhaustionConfirm^{buy}_e=
\mathbf{1}\{SpotCVDPush^{buy}_e\le \theta^{buy}_{cvd}\}
\cdot
\mathbf{1}\{SpotWhalePush^{buy}_e\le \theta^{buy}_{whale}\}
\]

统一新增输出字段：
- `spot_cvd_push_post_pivot`
- `spot_whale_push_post_pivot`
- `spot_continuation_risk`
- `spot_exhaustion_confirm`

**强度分数修订（固定）**
记原 `12.5` 分数为 \(score_e\equiv score\)。
\[
score'_e
=
score_e\cdot \big(1-\lambda_s\cdot SpotContinuationRisk^{buy}_e\big)
+ (1-score_e)\cdot 0.10\cdot SpotExhaustionConfirm^{buy}_e
\]
若仍沿用原字段名，可新增：
- `strength_score_xmk`
- `exhaustion_quality_score = score'_e`

- **必需数据类型（S）**：现货逐笔成交（bar 聚合 delta/relative_delta/cvd_7d），可选现货 bar OHLC

> **双市场修订（升级为双市场 CVD pack + 差分归因字段）**
> - 保留原容器结构思想（统一输出 `delta / relative_delta / delta_slant / cvd_7d`）
> - 新增 `futures + spot` 并行字段与 `xmk_*`（cross-market）差分归因字段

> 本定义将以下特征统一由 CVD 序列输出容器提供（避免重复计算、避免重复存储）：

> - `relative_delta`
> - `delta_slant`
> - `cvd_7d`

> 说明：
> - 这是**统一输出容器**（`cvd_pack_rolling_7d`），不是说数学上三者等同于 CVD。

### 输入

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）

### 计算公式

#### 14.1 bar 级 Delta（按输出周期 \(b\) 聚合）
设 \(I_b(t)\) 为属于 bar \(t\) 的逐笔成交索引集合，\(a_i\in\{+1,-1\}\) 表示主动方向（主动买=+1，主动卖=-1）：
\[
\Delta_b[t]=\sum_{i\in I_b(t)} a_i q_i
\]

#### 14.2 bar 级总成交量
\[
Vol_b[t]=\sum_{i\in I_b(t)} q_i
\]

> 若你同时维护 OHLCV，则 `volume` 字段就是 \(Vol_b[t]\)。

#### 14.3 Relative Delta（合并字段）
\[
RelativeDelta_b[t]=\frac{\Delta_b[t]}{Vol_b[t]+\varepsilon}
\]

其中 \(\varepsilon>0\) 用于避免分母为 0（实现中可取 `1e-12`）。

#### 14.4 Delta Slant（合并字段）
设 bar 的最高价、最低价分别为 \(H_b[t], L_b[t]\)，则：
\[
DeltaSlant_b[t]=\frac{\Delta_b[t]}{H_b[t]-L_b[t]+\varepsilon}
\]

实现约束（固定统一，避免歧义）：
- 当 \(H_b[t]-L_b[t] < min\_range\)（例如 `min_range = 1 * tick_size`）时：
  - 输出 `delta_slant = null`（固定），或
  - 用 `min_range` 替代分母做截断。
- 统一文档固定一种实现；若未特别说明，默认 **输出 `null`**。

#### 14.5 Rolling 7D CVD（核心字段）
\[
CVD^{7d}_b[t]=\sum_{u\in (t-7d,\ t]} \Delta_b[u]
\]

即：对当前 bar \(t\)，累计最近 7 天内同周期 bar 的 `delta`。

#### 14.6 Rolling 7D CVD 的递推更新（便于编码）
\[
CVD^{7d}_b[t]=CVD^{7d}_b[t-b]+\Delta_b[t]-\Delta_b[t-7d]
\]

其中：
- \(t-b\)：上一根同周期 bar
- \(t-7d\)：滚动窗口中被移出的同周期 bar 对应时间点（若不存在则视为 0）

初始化（前几根不足 7d 时）：
- 可直接求和：
\[
CVD^{7d}_b[t]=\sum_{u\in \mathcal{U}_t}\Delta_b[u]
\]
其中 \(\mathcal{U}_t=\{u\mid u\le t,\ u\text{ 为同周期bar}\}\)
- 或使用前缀和实现。

---

### 输出序列定义（7d 总长度）
设当前输出时刻为 \(T\)，输出周期为 \(b\)。定义该周期在最近 7 天内的 bar 时刻集合：
\[
\mathcal{T}_{b}^{7d}(T)=\{t_j \mid T-7d < t_j \le T,\ t_j \text{ 为周期 } b \text{ 的 bar 结束时刻}\}
\]

则输出序列为：
\[
Series^{CVDPack}_{b,7d}(T)=
\left\{
\big(
t,\ O_b[t],H_b[t],L_b[t],C_b[t],Vol_b[t],\Delta_b[t],RelativeDelta_b[t],DeltaSlant_b[t],CVD_b^{7d}[t]
\big)
\mid t\in\mathcal{T}_{b}^{7d}(T)
\right\}
\]

WhaleCount_w^{x}[t]=\sum_{i\in I_w^{x}(t)} Whale_i^{x}

WhaleSellCount_w^{x}[t]=\sum_{i\in I_w^{x}(t)} \mathbf{1}\{a_i=-1\}\cdot Whale_i^{x}
\]

#### 15.2 分市场大额交易量 / 名义金额 / Delta

WhaleNotionalSum_w^{x}[t]=\sum_{i\in I_w^{x}(t)} n_i^{(x)}\cdot Whale_i^{x}

WhaleDeltaQty_w^{x}[t]=\sum_{i\in I_w^{x}(t)} a_i q_i^{(x,ETH)}\cdot Whale_i^{x}

WhaleDeltaNotional_w^{x}[t]=\sum_{i\in I_w^{x}(t)} a_i n_i^{(x)}\cdot Whale_i^{x}
\]

WhaleDeltaNotionalGap_w^{S-F}[t]=WhaleDeltaNotional_w^{S}[t]-WhaleDeltaNotional_w^{F}[t]

{|WhaleDeltaNotional_w^{S}[t]|+|WhaleDeltaNotional_w^{F}[t]|+\varepsilon}

- `xmk_whale_delta_notional_gap_s_minus_f`

## 16) 资金费率（Funding Rate，窗口级状态 + 窗口内变更事件）

### 数据源要求（市场）
- **主计算市场**：合约（F）
- **现货数据（S）**：不需要（资金费率为合约专属）
- **必需数据类型**：合约 funding rate 时间序列/事件流 + 合约 mark price 时间序列（用于 `mark_price_last / mark_price_twap / mark_price_at_change`）

### 输入
- `funding_feed`
  - `timestamp = \tau_k`
  - `rate = fr_k`
- `mark_price_feed`（默认 tick/秒级；至少能做时间加权）
  - `timestamp = s_j`
  - `mark_price = m_j`
- 输出窗口
  - `w ∈ {15m, 1h, 4h, 1d, 3d}`

### 输出（示例结构）
> 每个窗口输出一条记录：既有**当前有效 funding**，也有**窗口内 funding 变更事件列表**，并记录 **mark 最新值 + mark_twap**。

```json
[
  {
    "ts":"2026-02-24T12:00:00Z",
    "window":"1h",

    "funding_current":0.000100,
    "funding_current_effective_ts":"2026-02-24T08:00:00Z",

    "funding_twa":0.000094,
    "mark_price_last":95688.3,
    "mark_price_last_ts":"2026-02-24T12:00:00Z",
    "mark_price_twap":95592.6,

    "change_count":1,
    "changes":[
      {
        "change_ts":"2026-02-24T08:00:00Z",
        "funding_prev":0.000075,
        "funding_new":0.000100,
        "funding_delta":0.000025,
        "mark_price_at_change":95380.2
      }
    ]
  }
]
```

### 计算公式（固定版）
设当前输出时刻为 \(T\)，窗口区间为：
\[
I_w(T)=(T-w,\ T]
\]

定义资金费率为阶梯函数（前向持有）：
\[
FR(t)=fr_k,\quad \tau_k\le t<\tau_{k+1}
\]

定义 mark 价格也用前向持有（LOCF）构造阶梯函数：
\[
M(t)=m_j,\quad s_j\le t<s_{j+1}
\]

#### 16.1 当前窗口结束时刻的有效资金费率（当前值）
\[
\tau^*(T)=\max\{\tau_k:\tau_k\le T\}
\]
\[
FundingCurrent_w[T]=FR(T)=fr(\tau^*(T))
\]

对应字段：
- `funding_current`
- `funding_current_effective_ts = \tau^*(T)`

#### 16.2 窗口内资金费率变更事件列表（changes）
定义窗口内发生的 funding 更新时刻集合：
\[
\mathcal{K}_w(T)=\{k\mid \tau_k\in I_w(T)\}
\]

对每个 \(k\in\mathcal{K}_w(T)\)（且存在前值 \(fr_{k-1}\)），定义变更事件：
\[
e_k=\Big(\tau_k,\ fr_{k-1},\ fr_k,\ \Delta fr_k\Big)
\]
其中：
\[
\Delta fr_k=fr_k-fr_{k-1}
\]

若要记录变更时 mark price，则取该时刻有效 mark：
\[
s^*(\tau_k)=\max\{s_j:s_j\le \tau_k\}
\]
\[
MarkAtChange(\tau_k)=M(\tau_k)=m(s^*(\tau_k))
\]

对应 `changes[]` 每项字段：
- `change_ts = \tau_k`
- `funding_prev = fr_{k-1}`
- `funding_new = fr_k`
- `funding_delta = \Delta fr_k`
- `mark_price_at_change = MarkAtChange(\tau_k)`

并且：
\[
ChangeCount_w[T]=|\mathcal{K}_w(T)|
\]

#### 16.3 窗口内资金费率时间加权均值（Funding TWA）
资金费率在窗口内按时间加权平均：
\[
FundingTWA_w[T]=\frac{1}{w}\int_{I_w(T)} FR(t)\,dt
\]

离散实现（阶梯函数精确积分）：
设窗口内及边界相关 funding 分段为 \([u_r,u_{r+1})\)，该段 funding 常数为 \(c_r\)，则：
\[
FundingTWA_w[T]=\frac{1}{w}\sum_r c_r\cdot (u_{r+1}-u_r)
\]

对应字段：
- `funding_twa`

#### 16.4 窗口结束时刻的 mark 最新值（当前值）
\[
s^*(T)=\max\{s_j:s_j\le T\}
\]
\[
MarkLast_w[T]=M(T)=m(s^*(T))
\]

对应字段：
- `mark_price_last`
- `mark_price_last_ts = s^*(T)`

#### 16.5 窗口内 mark 价格时间加权均值（Mark TWAP）
\[
MarkTWAP_w[T]=\frac{1}{w}\int_{I_w(T)} M(t)\,dt
\]

离散实现（阶梯函数精确积分）：
设窗口内及边界相关 mark 分段为 \([v_r,v_{r+1})\)，该段 mark 常数为 \(d_r\)，则：
\[
MarkTWAP_w[T]=\frac{1}{w}\sum_r d_r\cdot (v_{r+1}-v_r)
\]

对应字段：
- `mark_price_twap`

## 17) Trade Flow Toxicity (VPIN)

### 数据源要求（市场）
- **主输出市场**：合约（F）
- **现货数据（S）**：需要（输出 `VPIN_fut` 与 `VPIN_spot` 并计算毒性差）
- **必需数据类型（F/S）**：逐笔成交（价格、数量、主动方向），按 volume bucket 聚合

> **双市场修订（VPIN_fut vs VPIN_spot）**
> - VPIN 仍非第一优先级，但做成双市场后可观察“毒性来源差异”

### 输入
- `trades_futures`（逐笔，主动方向）
- `trades_spot`（逐笔，主动方向）
- volume bucket size（统一按市场分别配置）
  - `V_F^*`, `V_S^*`
- rolling bucket count `n`
- bucket-to-time 映射规则（用于输出到时间序列）
- （固定）单位模式
  - `vpin_unit ∈ {eth_qty, usdt_notional}`（跨市场对比时统一固定同一种）

### 输出字段
- 沿用本指标定义的结构化字段（本版省略示例 JSON，仅保留公式与字段定义）

### 计算公式

对 \(x\in\{F,S\}\) 分别按固定成交量桶计算。第 \(j\) 个桶的买量/卖量为 \(V_{j}^{B,x},V_{j}^{S,x}\)，满足：
\[
V_x^*=V_{j}^{B,x}+V_{j}^{S,x}
\]

桶级 VPIN：
\[
VPIN^{x}[j]=\frac{1}{nV_x^*}\sum_{m=j-n+1}^{j}\left|V_{m}^{B,x}-V_{m}^{S,x}\right|
\]

映射到时间窗口输出（对每个市场独立）：
- 取窗口末最后一个桶值：
\[
VPIN_w^{x}[t]=VPIN^{x}[j_t^{x}]
\]
- 或窗口内平均：
\[
VPIN_w^{x}[t]=\frac{1}{|\mathcal{J}_w^{x}(t)|}\sum_{j\in\mathcal{J}_w^{x}(t)}VPIN^{x}[j]
\]

#### 17.1 跨市场毒性差（新增）
\[
VPINGap_w^{S-F}[t]=VPIN_w^{S}[t]-VPIN_w^{F}[t]
\]
\[
VPINRatio_w^{S/F}[t]=\frac{VPIN_w^{S}[t]}{VPIN_w^{F}[t]+\varepsilon}
\]

（固定）标准化差值：
\[
zVPIN_w^{x}[t]=\frac{VPIN_w^{x}[t]-\mu^{vpin,x}[t]}{\sigma^{vpin,x}[t]+\varepsilon}
\]
\[
zVPINGap_w^{S-F}[t]=zVPIN_w^{S}[t]-zVPIN_w^{F}[t]
\]

## 18) avwap（锚定 VWAP，7d 总长度，按窗口输出序列）

### 数据源要求（市场）
- **主输出市场**：合约（F）
- **现货数据（S）**：需要（作为现金市场锚，输出 `AVWAP_spot` 与跨市场偏离）
- **必需数据类型（F/S）**：逐笔成交（优先）；若缺失可使用各自 OHLCV 近似
- **可选增强数据（F）**：合约 mark price 时间序列（若输出 `price_minus_spot_avwap_futmark`）

> **双市场修订（futures AVWAP + spot AVWAP 现金市场锚）**
> - 保留 `futures AVWAP` 作为主交易执行/衍生品锚
> - 新增 `spot AVWAP` 作为现金市场锚，用于偏离/回归与真假突破过滤

### 输入
- `trades_futures`（逐笔）或 `OHLCV_futures`（近似）
- `trades_spot`（逐笔）或 `OHLCV_spot`（近似）
- `anchor_time = \tau`
- `lookback = 7d`（固定）
- `output_window ∈ {15m, 1h, 4h, 1d, 3d}`（输出采样间隔）
- （固定）`price_basis_mode ∈ {raw, basis_adjusted}`（跨市场比较时可选）

### 输出（示例结构）
> 输出统一升级为双市场序列（同一采样点同时给出 futures/spot AVWAP 与差值字段）。

```json
{
  "indicator":"avwap_dual_market",
  "anchor_ts":"2026-02-17T00:00:00Z",
  "lookback":"7d",
  "window":"1h",
  "series":[
    {
      "ts":"2026-02-17T01:00:00Z",
      "avwap_fut":95420.3,
      "avwap_spot":95408.9,
      "xmk_avwap_gap_f_minus_s":11.4,
      "price_minus_spot_avwap_futmark":35.2
    }
  ]
}
```

### 计算公式

#### 18.1 双市场逐笔版 AVWAP（核心定义）
对 \(x\in\{F,S\}\)，任意时刻 \(t>\tau\)：
\[
AVWAP^{x,(\tau)}[t]=
\frac{\sum_{i:\tau<t_i^x\le t} p_i^x q_i^{(x,ETH)}}
{\sum_{i:\tau<t_i^x\le t} q_i^{(x,ETH)}+\varepsilon}
\]

其中：
- \(x=F\)：futures
- \(x=S\)：spot

#### 18.2 双市场 7d 输出序列（按采样窗口）
设采样间隔 \(w\)，当前输出时刻为 \(T\)，采样点集合：
\[
\mathcal{T}_{w}^{7d}(T)=\{t_j \mid T-7d < t_j \le T,\ t_j \text{按间隔 } w \text{采样}\}
\]

则双市场输出序列：
\[
Series^{dualAVWAP}_{w,7d}(T)=
\left\{
\big(t,\ AVWAP^{F,(\tau)}[t],\ AVWAP^{S,(\tau)}[t]\big)\mid t\in\mathcal{T}_{w}^{7d}(T)
\right\}
\]

#### 18.3 跨市场 AVWAP 差分字段（新增）
双 AVWAP 差值：
\[
AVWAPGap^{F-S}[t]=AVWAP^{F,(\tau)}[t]-AVWAP^{S,(\tau)}[t]
\]

若使用合约价格（如 mark / last）对照现金市场锚：
\[
PriceMinusSpotAVWAP^{F}[t]=P^{F}[t]-AVWAP^{S,(\tau)}[t]
\]

（固定）归一化版本：
\[
zAVWAPGap[t]=\frac{AVWAPGap^{F-S}[t]}{\sigma^{gap}_{w}[t]+\varepsilon}
\]

> 解释：
> - `AVWAP^S`：现金市场锚（你关心的“现货主导是否持续”）
> - `PriceMinusSpotAVWAP^F`：合约价格相对现金锚的偏离程度（可用于假突破/回归过滤）

#### 18.4 递推实现（双市场）
对每个市场 \(x\in\{F,S\}\) 分别维护累计量：
\[
N^x[t]=\sum_{i:\tau<t_i^x\le t} p_i^x q_i^{(x,ETH)},\qquad
D^x[t]=\sum_{i:\tau<t_i^x\le t} q_i^{(x,ETH)}
\]
\[
AVWAP^{x,(\tau)}[t]=\frac{N^x[t]}{D^x[t]+\varepsilon}
\]

逐笔更新：
\[
N^x[t_k]=N^x[t_k^-]+p_k^x q_k^{(x,ETH)},\qquad
D^x[t_k]=D^x[t_k^-]+q_k^{(x,ETH)}
\]

#### 18.5 OHLCV 近似版（双市场）
若某市场只有 bar 数据（周期 \(b\)），则对该市场使用典型价近似：
\[
\tilde p_b^{x}[t]=\frac{H_b^{x}[t]+L_b^{x}[t]+C_b^{x}[t]}{3}
\]
\[
AVWAP_b^{x,(\tau)}[t]\approx
\frac{\sum_{u:\tau<u\le t}\tilde p_b^{x}[u]\cdot Vol_b^{x}[u]}
{\sum_{u:\tau<u\le t}Vol_b^{x}[u]+\varepsilon}
\]

## 20) tpo_market_profile（TPO 时间型 Market Profile，含 IB + Single Prints）

### 数据源要求（市场）
- **主计算市场**：合约（F）
- **现货数据（S）**：不需要（可选用于“现金市场对照”，但本指标默认只算 F）
- **必需数据类型**：OHLCV bar（推荐最细粒度 `1m`）；若有逐笔成交可用于更细的价格分箱，但不是必需

### 目标
- 在每个会话（session）内，用 **Time-at-Price**（价格被“停留/覆盖”的时间）构建轮廓
- 输出 **TPO POC/VAH/VAL** 与 **单印区（Single Prints）**、**初始平衡区间（IB）**
- （可选）输出 **Developing POC** 的时间序列，用于观察会话内关键位演化

### window（硬约束）
- **会话窗口（session 粒度）**：`w_session ∈ {4h, 1d}`
- **IB 窗口**：`w_ib = 1h`（每个 `1d session` 的前 1 小时；若 `w_session=4h`，则取该 4h 的前 1 小时）
- **输出采样（developing 系列）**：`w_out ∈ {15m, 1h}`
- **价格分箱**：`rows_nb = 64`（默认；可调），`value_area_pct = 0.70`

### 输出（示例结构）
```json
{
  "indicator":"tpo_market_profile",
  "session_window":"1d",
  "session_start":"2026-03-05T00:00:00Z",
  "rows_nb":64,
  "value_area_pct":0.70,
  "tpo_poc":95380.5,
  "tpo_vah":95820.0,
  "tpo_val":95040.0,
  "initial_balance_high":95610.0,
  "initial_balance_low":95210.0,
  "tpo_single_print_zones":[
    {"low":94880.0,"high":94960.0,"score":1}
  ],
  "dev_series":[
    {"ts":"2026-03-05T03:00:00Z","tpo_dev_poc":95420.0,"tpo_dev_vah":95680.0,"tpo_dev_val":95240.0}
  ]
}
```

### 计算方式（核心）
#### 20.1 价格分箱
对一个 session（起止 \([T_0,T_1]\)），定义价格范围：
\[
P_{min}=\min_{t\in[T_0,T_1]} L[t],\quad
P_{max}=\max_{t\in[T_0,T_1]} H[t]
\]
按 `rows_nb` 分箱，箱宽：
\[
\Delta=\frac{P_{max}-P_{min}}{rows\_nb}
\]
第 \(k\) 个箱的中心价 \(b_k=P_{min}+(k+0.5)\Delta\)。

#### 20.2 TPO 计数（Time-at-Price）
用 1m bar 近似时间占用：对每根 1m bar（高低 \([L[t],H[t]]\)），将其覆盖到的所有价格箱计数 +1：
\[
score_k = \sum_{t\in session} \mathbf{1}\big([L[t],H[t]]\cap bin_k 
eq \emptyset\big)
\]
（若使用逐笔成交可将“覆盖”替换为“成交落入箱内”，但默认采用 bar 覆盖法。）

#### 20.3 TPO POC / Value Area（VAH/VAL）
- **TPO POC**：得分最高的箱中心价：
\[
POC = b_{\arg\max_k score_k}
\]
- **Value Area**：以 POC 为中心向两侧扩展，直到累计得分达到 `value_area_pct`：
\[
\frac{\sum_{k\in VA} score_k}{\sum_k score_k} \ge value\_area\_pct
\]
得到 VAH/VAL 为 VA 内最高/最低箱边界（或中心价）。

#### 20.4 Single Prints（单印区）
单印区（弱接受/拒绝）用“仅被覆盖 1 次”的价格箱近似：
\[
SinglePrint = \{k \mid score_k = 1\}
\]
将连续箱合并为区间，输出到 `tpo_single_print_zones[]`。

#### 20.5 Initial Balance（IB）
IB 为 session 起始 `w_ib` 时段内价格区间：
\[
IB_{high}=\max_{t\in[T_0,T_0+w_{ib}]} H[t],\quad
IB_{low}=\min_{t\in[T_0,T_0+w_{ib}]} L[t]
\]

#### 20.6 Developing POC（可选序列）
在 session 内按 `w_out` 取锚点 \(u\)，仅使用 \([T_0,u]\) 的数据重复 19.2~19.3，得到 `tpo_dev_poc / tpo_dev_vah / tpo_dev_val` 序列。


## 21) rvwap_sigma_bands（Rolling VWAP + σ Bands + 偏离 z-score）

### 数据源要求（市场）
- **主输出市场**：合约（F）
- **现货数据（S）**：不需要（可在上层做“对照/归因”，本指标默认只算 F）
- **必需数据类型**：逐笔成交（优先）或 OHLCV（近似）

### window（硬约束）
- **滚动窗口**：`w ∈ {15m, 4h, 1d}`
- **内部更新粒度**：`1m`（内部每分钟更新一次滚动统计）
- **对外输出采样**：`w_out ∈ {15m, 1h}`（对外下采样展示）

### 输出字段（核心）
- `rvwap_w`：滚动 VWAP
- `rvwap_sigma_w`：窗口内加权标准差
- `rvwap_band_{+1,+2,-1,-2}`：±1σ/±2σ 轨道
- `z_price_minus_rvwap`：价格相对 RVWAP 的 z 化偏离

### 计算公式
#### 21.1 滚动 VWAP（逐笔版）
设窗口 \(w\)，对任意时刻 \(t\)：
\[
RVWAP_w[t]=
\frac{\sum_{i:t-w<t_i\le t} p_i q_i^{(ETH)}}
{\sum_{i:t-w<t_i\le t} q_i^{(ETH)}+\varepsilon}
\]

#### 21.2 加权标准差与 Bands
加权方差：
\[
Var_w[t]=
\frac{\sum_{i:t-w<t_i\le t} q_i^{(ETH)}(p_i-RVWAP_w[t])^2}
{\sum_{i:t-w<t_i\le t} q_i^{(ETH)}+\varepsilon}
\]
\[
\sigma_w[t]=\sqrt{Var_w[t]}
\]
Bands：
\[
Band_{\pm k}[t]=RVWAP_w[t]\pm k\cdot\sigma_w[t],\quad k\in\{1,2\}
\]

#### 21.3 偏离 z-score
\[
z\_price\_minus\_rvwap[t]=\frac{P[t]-RVWAP_w[t]}{\sigma_w[t]+\varepsilon}
\]

#### 21.4 OHLCV 近似版（当逐笔缺失）
在 bar 周期 \(b\) 上，用典型价近似逐笔价格：
\[
\tilde p_b[t]=\frac{H_b[t]+L_b[t]+C_b[t]}{3}
\]
\[
RVWAP_{w}[t]\approx
\frac{\sum_{u:t-w<u\le t}\tilde p_b[u]\cdot Vol_b[u]}
{\sum_{u:t-w<u\le t}Vol_b[u]+\varepsilon}
\]
（标准差同理用 \(Vol_b\) 做权重。）


## 22) high_volume_pulse（高量能脉冲 z-score + intrabar POC）

### 数据源要求（市场）
- **主输出市场**：合约（F）
- **现货数据（S）**：不需要
- **必需数据类型**：
  - `1m` OHLCV（用于量能统计与脉冲判定）
  - 逐笔成交（用于 `intrabar_poc_*`；若缺失则不输出 intrabar POC）

### window（硬约束）
- **量能 z-score 窗口**：`w_z ∈ {1h, 4h, 1d}`
- **intrabar POC 计算粒度**：`1m`
- **可选汇总输出**：在 `{15m, 1h}` 上输出“该窗口内最大 intrabar_poc_volume 对应价位”

### 输出字段（核心）
- `volume_spike_z_w`：量能脉冲 z-score（对应窗口 \(w_z\)）
- `is_volume_spike_z2 / is_volume_spike_z3`：事件标记（≥2σ / ≥3σ）
- `intrabar_poc_price` / `intrabar_poc_volume`：单根 1m bar 内的 POC
- 语义约束：
  - 顶层 `intrabar_poc_*` 仅表示 current `1m` sidecar
  - 持仓后判断优先读取 `by_z_window` 与 `intrabar_poc_max_by_window`

### 计算方式
#### 22.1 量能脉冲 z-score（基于 1m volume 聚合）
先把 `1m` 成交量聚合到目标窗口 \(w_z\)：
\[
Vol_{w_z}[t]=\sum_{u\in(t-w_z,t]} Vol_{1m}[u]
\]
均值与标准差（可用 SMA 或 EMA，默认 SMA）：
\[
\mu_{w_z}[t]=SMA(Vol_{w_z}, w_z),\quad
\sigma_{w_z}[t]=STDEV(Vol_{w_z}, w_z)
\]
z-score：
\[
Z^{vol}_{w_z}[t]=\frac{Vol_{w_z}[t]-\mu_{w_z}[t]}{\sigma_{w_z}[t]+\varepsilon}
\]
事件：
\[
is\_spike\_{z2}[t]=\mathbf{1}(Z^{vol}_{w_z}[t]\ge2),\quad
is\_spike\_{z3}[t]=\mathbf{1}(Z^{vol}_{w_z}[t]\ge3)
\]

#### 22.2 intrabar POC（单根 1m bar 内成交量控制价位）
对某根 1m bar 的逐笔集合 \(\mathcal{I}_{1m}\)，按价格步长（如 tick 或自定义 bin）聚合成交量：
\[
V(p)=\sum_{i\in\mathcal{I}_{1m},\ p_i\in bin(p)} q_i
\]
\[
POC_{1m}=\arg\max_p V(p),\quad
intrabar\_poc\_volume=\max_p V(p)
\]

#### 22.3 15m/1h 汇总（可选）
在窗口 \(W\in\{15m,1h\}\) 内，取 intrabar POC “成交量最大”的那根 1m bar：
\[
POC^{maxV}_{W} = POC_{1m^*},\quad
1m^*=\arg\max_{u\in W} intrabar\_poc\_volume[u]
\]


## 23) ema_trend_regime（EMA 趋势带 + 多周期 EMA100/200 Regime）

### 数据源要求（市场）
- **主输出市场**：合约（F）
- **现货数据（S）**：不需要
- **必需数据类型**：OHLCV（至少 close），支持多周期 bar

### window（硬约束）
- **基础计算周期**：`tf_base = 1m`（计算 EMA13/21/34 趋势带）
- **高周期趋势锚**：`tf_htf ∈ {4h, 1d}`（计算 EMA100/200 并下采样/前向填充到低周期）
- **对外输出采样**：`w_out ∈ {15m, 1h}`

### 输出字段（核心）
- `ema_13 / ema_21 / ema_34`
- `ema_band_low / ema_band_high`（趋势带：min/max）
- `ema_100_htf / ema_200_htf`（在 `4h` 与 `1d` 上分别输出一组）
- `trend_regime`（可选：bull/bear/neutral）
- 语义约束：
  - `ema_13 / ema_21 / ema_34 / ema_band_*` 仅表示 current `1m` sidecar
  - 持仓后判断优先读取 `trend_regime_by_tf` 与按 `w_out` 输出的前向填充序列

### 计算公式
#### 23.1 EMA 定义
对周期 \(n\)：
\[
EMA_n[t]=\alpha C[t]+(1-\alpha)EMA_n[t-1],\quad \alpha=\frac{2}{n+1}
\]

#### 23.2 趋势带（13/21/34）
\[
Band_{low}[t]=\min(EMA_{13}[t],EMA_{21}[t],EMA_{34}[t])
\]
\[
Band_{high}[t]=\max(EMA_{13}[t],EMA_{21}[t],EMA_{34}[t])
\]

#### 23.3 Regime（基于高周期 EMA100/200）
在高周期 \(TF\in\{4h,1d\}\) 上：
\[
Regime_{TF}[t]=
\begin{cases}
bull,& EMA_{100}^{TF}[t]>EMA_{200}^{TF}[t]\\
bear,& EMA_{100}^{TF}[t]<EMA_{200}^{TF}[t]\\
neutral,& \text{otherwise}
\end{cases}
\]
下采样到低周期采用“前向填充”，用于你后续信号的 regime filter。

### 2.2.2 事件触发条件（建议直接写进采集器状态机）
> 这部分是为了让 REST 真正“补洞化”，避免退化成轮询。

#### A) Order Book 重建触发（Spot/Futures 共用逻辑）
- `depth` 增量链断裂（例如 futures 文档要求 `pu == prev_u`，不满足则重建）
- 重连后首次启动本地簿
- 本地簿校验失败（价差异常、负深度、排序异常等）
- 处理延迟过大，确认增量队列已不可追

**动作**
1. 缓冲当前 `@depth` 增量事件
2. 请求 `GET /depth` 快照
3. 按 Binance 文档规则对齐 `lastUpdateId / U / u / pu`
4. 切回增量流持续更新

#### B) `aggTrade` 补洞触发（Spot/Futures）
- WS 重连
- 发现 trade id / aggregate id 非连续（按你实现的连续性规则）
- 消费者 backlog 超阈值导致落后
- 指标侧检测到时间窗中成交为空但 bookTicker/depth 明显活跃（异常）

**动作**
- 调 `GET /aggTrades` 按 `fromId`（优先）或时间范围拉取缺失段
- 回放缺失段后再恢复实时流
- 补洞期间对下游指标打 `backfill_in_progress` 标记（避免误报）

#### C) Funding/Mark 初值触发（Futures）
- `markPrice@1s` 订阅后 N 秒内未收到首包（例如 2–3 秒）
- WS 重连后资金/mark 状态为空
- 资金费率窗口对账发现缺失 funding change 事件

**动作**
- 先调用 `GET /fapi/v1/premiumIndex` 初始化当前 `mark + funding`
- 再用 `GET /fapi/v1/fundingRate` 回填历史变更（仅 `16` 指标）

#### D) Kline 补洞触发（本版重点：多周期直存；`3d` 本地聚合）
- `futures.kline_{1m|15m|1h|4h|1d}` 任一 WS 重连后出现 bar 缺口
- 收到的 Kline 序列在对应周期边界不连续（缺 bar）
- 指标 `3/6/7/12/13` 计算窗口检测到 `1m OHLCV` 缺失
- 指标 `14/18` 使用的 `15m/1h/4h/1d` Kline 缺失或状态不可信
- Kline WS 延迟/丢包导致当前或上一个 bar 状态不可信

**动作**
- 调 `GET /fapi/v1/klines` 按缺失时间段回补 `1m/15m/1h/4h/1d`（按具体缺口周期请求）
- 回放缺失 Kline 后再释放依赖 OHLCV 的指标计算
- 回补期间对依赖 Kline 的指标打 `bar_backfill_in_progress` 标记（避免事件误触发）
- `3d` bar 不从交易所拉取：由本地聚合生成（优先基于 `1d`，必要时基于 `1m` 重建）

# 19) kline_history

### 数据源要求（市场）
- 主计算市场：futures + spot
- 必需数据类型：Indicator Engine 内存中的已完成 1 分钟历史窗口
- 输出方式：作为新的分钟快照指标进入 `ind.snapshot` 和 `ind.minute_bundle`

### 目标
- 在每个 `1m` 指标快照里附带多周期历史 K 线
- 默认输出：
  - `1m`: 最近 `1024` 条
  - `15m`: 最近 `120` 条
  - `1h`: 最近 `120` 条
  - `1d`: 最近 `120` 条
- 若当前历史不足，则返回全部已缓存数据

### 输出字段
- `intervals.1m.markets.futures.bars[]`
- `intervals.1m.markets.spot.bars[]`
- `intervals.15m.markets.futures.bars[]`
- `intervals.15m.markets.spot.bars[]`
- `intervals.1h.markets.futures.bars[]`
- `intervals.1h.markets.spot.bars[]`
- `intervals.1d.markets.futures.bars[]`
- `intervals.1d.markets.spot.bars[]`

每根 bar 输出：
- `open_time`
- `close_time`
- `open`
- `high`
- `low`
- `close`
- `volume_base`
- `volume_quote`
- `is_closed`
- `minutes_covered`
- `expected_minutes`

### 聚合定义
设 1 分钟历史序列为 `M={m_t}`，每个 `m_t` 含：
- `open_t`
- `high_t`
- `low_t`
- `close_t`
- `volume_base_t`
- `volume_quote_t`

对任意周期 `W ∈ {1m, 15m, 1h, 1d}`，定义该周期 K 线为：

`Open_W =` 该窗口内第一根分钟 bar 的开盘价  
`High_W = max(high_t)`  
`Low_W = min(low_t)`  
`Close_W =` 该窗口内最后一根分钟 bar 的收盘价  
`VolumeBase_W = Σ volume_base_t`  
`VolumeQuote_W = Σ volume_quote_t`

### 闭合定义
- `1m` bar 恒为 `is_closed=true`
- 对 `15m/1h/1d`：
  - 若 `close_time <= 当前快照时刻对应的分钟收盘时间`，则 `is_closed=true`
  - 否则 `is_closed=false`

### 说明
- 该指标用于给 LLM / 策略侧直接提供近期多周期 OHLCV 上下文
- `1d` 默认采用“近期分钟历史聚合 + 数据库 `md.kline_bar(interval_code='1d')` 补旧日线”的方式补齐
- 因此最近几天的日线保持与当前分钟状态一致，较旧的已收盘日线从数据库回补到目标条数
